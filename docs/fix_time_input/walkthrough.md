# 不正・不適切な時間の入力ブロック 修正内容の確認

シミュレーションレポートで指摘された「2. 不正・不適切な時間の入力」について修正を行いました。

## 実施内容

### `timeOverlapWarnings` ロジックの強化
- **同一現場内の重複検知:** 従来は除外されていた、同一現場内の作業項目どうしでの時間重複を検知できるようにしました。
- **逆転時間の検知:** 終了時間が開始時間以前になっている場合（例: 開始17:00、終了08:00）、専用の警告を表示するようにしました。
- **メッセージの改善:** 重複が発生した際に、どの作業項目やどの現場で発生したかを正確にメッセージに表示するようにしました。

### `handleSubmit` への送信ブロック追加
- 上記のような時間重複や逆転の警告が出ている場合、「今日の実績を送信」ボタンを押しても、エラーのToast通知を表示し**処理を中断（送信ブロック）**する仕組みを追加しました。

## 確認事項
- [x] 同一現場内で時間が被る入力をした場合、赤い警告が表示されること。
- [x] 警告メッセージに「作業項目A」と「作業項目B」のように具体的な名前が表示されること。
- [x] 開始時間と終了時間が逆転した入力をした場合、赤い警告が表示されること。
- [x] 警告が表示された状態で送信ボタンを押すと、エラーがポップアップし、データが送信されないこと。
