# 不正・不適切な時間の入力ブロック 実装計画

シミュレーションレポートで指摘された「2. 不正・不適切な時間の入力」を修正し、日報の入力ミスを未然に防ぐ仕組みを導入します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> 修正後、時間の重複（他現場間、あるいは同一現場内の作業項目間）や、開始時間と終了時間が逆転しているなどの警告が出ている状態では、**「今日の実績を送信」ボタンを押しても送信がブロック**されるようになります。

## 提案される変更

### WorkerApp Component

#### [MODIFY] [WorkerApp.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/WorkerApp.jsx)

- **`timeOverlapWarnings`のロジック改修**
  1. 従来はチェックから除外（`continue`）されていた「同一現場内の異なる作業時間」についても重複チェックを行うよう修正します。
  2. 開始時間と終了時間が逆転している、または同一（例: 17:00開始、08:00終了）の場合に、「終了時刻が開始時刻以前になっています」という警告を表示する機能を追加します。
  3. 重複時の警告メッセージを、同一現場内の重複の場合は「作業項目名」、別現場の場合は「現場名」を出力するようにし、どこで重複しているかをユーザーが判別しやすくします。

- **`handleSubmit`の送信ブロック処理追加**
  1. `timeOverlapWarnings.length > 0` の場合、エラーのトースト通知（`showToast`）を表示し、送信処理を中断するガード節（`return`）を追加します。

## 検証プラン

### 手動確認
1. WorkerApp画面でログインし、任意の現場を選択する。
2. 同一現場内の2つの作業項目で、時間が重なるように（例: 08:00~10:00, 09:00~11:00）入力する。
   - 赤い警告の枠が表示され、作業項目名で重複内容が示されるか確認する。
   - 送信ボタンを押して、「時間の重複または不整合の警告が出ています」と表示され送信が防がれることを確認する。
3. 開始時間と終了時間を逆転させて（例: 開始17:00、終了08:00）入力する。
   - 警告が表示されることを確認し、同じく送信がブロックされるか確認する。
