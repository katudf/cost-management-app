# ウォークスルー: データ取得フェッチの最適化

本対応では、初期起動時のデータ読み込み速度向上のため、完了した現場の詳細データ（タスク、日報レコード、外注レコード）のフェッチを遅延ロード化し、それに伴う React クラッシュバグの解消を行いました。

## 変更内容

### 1. `useSupabaseData.js` の修正
* **初期ロードの最適化**:
  * `fetchAllData` にて完了済み（`COMPLETED`）現場の詳細データをクエリから除外し、初期フェッチのペイロードを削減しました。
* **遅延ロードの初期値設定（クラッシュバグの解消）**:
  * 未ロードの現場データオブジェクトにおいて、`masterData`, `records`, `subcontractors` を `null` ではなく空の配列（`[]`）として定義し、`progressData` を空オブジェクト（`{}`）として初期化しました。
  * これにより、初期化時に `useDashboardStats` 等が `null` 参照による `Cannot read properties of null (reading 'map')` エラーで React がクラッシュする現象を完全に防止しました。
  * `_isLoaded` フラグを新規に導入し、未ロードの現場は `_isLoaded: false`、ロード済みの現場は `_isLoaded: true` と判定できるようにしました。
* **遅延フェッチ処理**:
  * `fetchProjectDetails` にて、非同期フェッチ完了後に `_isLoaded: true` を付与して `projects` ステートを部分更新するよう修正しました。

### 2. `AdminApp.jsx` の修正
* **トリガー条件の更新**:
  * 現場変更時の詳細データ遅延ロードトリガーを `activeProj.masterData === null` から `activeProj._isLoaded === false` に変更し、型安全かつ明示的なフラグでの遅延ロード判定に改善しました。

1. **ホームページのクラッシュ回避**:
