# スマホアプリ画面の右切れ修正 - 実装計画

## 概要
作業員向けスマホアプリ（`WorkerApp.jsx`）で、画面幅が狭い場合（スマホ等）に「今日作業した時間」の `+` ボタンが右に押し出されて見えなくなってしまう問題（または画面全体が横スクロールしてしまう問題）を修正します。

## 原因と修正内容
- **原因**: 
  Flexbox（`flex items-center gap-2`）の中に配置された `<input type="number">` 要素にはブラウザ特有のデフォルトの最小幅が設定されています。十分な幅がないスマホ画面の場合、`flex-1` が指定されていてもデフォルト幅のせいで縮小しきれず、隣の `+` ボタンがコンテナの外（画面右外）に押し出されてしまいます。
- **修正内容 (`src/WorkerApp.jsx`)**:
  - `input` タグのクラスに `w-full min-w-0` (または `w-0`) を追加し、Flexbox内での柔軟な縮小を許可します。
  - 具体的には `className="flex-1 h-14 text-center..."` を `className="flex-1 w-full min-w-0 h-14 text-center..."` に変更します。

## 検証計画
- `npm run dev` でローカルサーバーを起動し、モバイルビューでの表示を確認します。
- `+` ボタンが画面内に収まり、画面全体が横スクロールしなくなっていることを確認します。
