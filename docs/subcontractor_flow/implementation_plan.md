# 常用下請け業者処理フロー 実装計画

## 背景・目的
常用で下請け（協力）業者を使った場合、現在の紙ベースの作業報告書には「協力業者名」と「人数」を職長が記入している。
アプリ上でも同様に職長がこれらの情報を入力でき、かつプロジェクトのコスト管理（1人工あたり25,000円）に自動で組み込まれるようにシステムを拡張する。

## 課題への対応方針
> 協力業者の作業時間は作業項目に割り振っていないので、コスト計算をどのように扱うか？
これに対する提案：
協力業者のコスト（人数 × 25,000円）は、特定の「作業項目（タスク）」に紐付けるのではなく、「プロジェクト（現場）全体にかかった間接コスト」または「協力会社費」として扱う仕様とします。
管理アプリ（`AdminApp.jsx`）でのプロジェクト原価（総コスト）計算時には、「自社作業員のタスク別労務費の合計」に、「協力会社費」を単純合算する形とします。

## 提案する実装内容（Proposed Changes）

### 1. データベース (Supabase)
協力業者の日々の記録を管理するための新しいテーブル `SubcontractorRecords` を作成します。
(または要件の都合上、既存テーブルにカラムを追加するアプローチを取るかを選択可能ですが、複数業者の記録を入れるため別テーブルが推奨です)

*   **[NEW] `SubcontractorRecords` テーブル**
    *   `id`: uuid (Primary Key)
    *   `project_id`: integer (Projects テーブルへの外部キー)
    *   `date`: date (記録日)
    *   `company_name`: text (協力会社名)
    *   `worker_count`: numeric/integer (人数)
    *   `unit_price`: numeric (1人工あたりの金額。アプリ側から 25,000 として送信、またはテーブルデフォルト値)
    *   `worker_name`: text (入力した職長名など)

### 2. WorkerApp.jsx (職長アプリ側の改修)
*   **[MODIFY] `src/WorkerApp.jsx`**
    *   状態管理に「協力業者リスト `subcontractors`」を追加。
    *   画面の「今日作業した時間」セクションの下（または上）に、新しいセクションとして「協力業者の入力（職長のみ、または全員）」のUIを追加。
    *   「会社名」のテキスト入力と、「人数」の数値入力を実装。行を追加・削除できるようにする。
    *   `handleSubmit` (保存処理) にて、入力された協力会社データを `SubcontractorRecords` テーブルに保存・更新するロジックを追加。

### 3. AdminApp.jsx (管理アプリ側の改修)
*   **[MODIFY] `src/AdminApp.jsx`**
    *   データフェッチ処理 (`useEffect`) にて、`SubcontractorRecords` データもSupabaseから取得。
    *   コスト計算ロジック（プロジェクト詳細や月間集計など）において、取得した `SubcontractorRecords` から「人数 × 25000円」を計算し、従来のタスク別労務費と合算して「総コスト（原価）」として表示するように修正。
    *   （必要に応じて）「施工管理シート」等に「本日の協力業者」を表示。

## 検証計画 (Verification Plan)
### 手動検証 (Manual Verification)
1.  **テーブル作成の確認**:
    *   Supabase SQL エディターまたは設定画面にて、`SubcontractorRecords` テーブルが作成されたことを確認する。
2.  **WorkerAppでの操作と保存**:
    *   ブラウザでWorkerAppを開き、既存の職長ユーザーでログイン。
    *   現場を選択し、協力会社情報を入力して保存を行う。
    *   Supabase にデータが保存されたことを確認する。
3.  **AdminAppでのコスト反映確認**:
    *   管理画面（AdminApp）を開き、対象プロジェクトの詳細を開く。
    *   協力業者の「人数 × 25,000円」がコストとして正しく全体に加算・表示されていることを確認する。

## User Review Required
> [!IMPORTANT]
> - 下請け業者のコストを「タスクに紐づかない全体のコスト」として加算する方針でよろしいでしょうか。
> - データベース設計として、新しいテーブル `SubcontractorRecords` を作成する方針で進めてよいかご確認ください。（既存の `DailyReports` なしで `TaskRecords` が日報の役割を果たしているため、別テーブルにするのが最も安全で集計しやすいです。）
> - WorkerApp にて、これらを入力できるのは「職長権限を持つユーザーのみ」とするか、どの現場でも作業員全員が入力可能とするべきか、ご意見があれば教えてください。（一旦、「現場に割り当てられた職長」のみ入力可能なUIとし、必要に応じて調整します。）
