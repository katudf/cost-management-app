# 複数現場切り替え機能の確認 (Walkthrough)

## 変更内容
- `src/App.jsx` 内のデータ管理構造を、既存の単一フラット状態から、`projects` 配列と `activeProjectId` を用いた複数現場管理構造に刷新しました。
- アプリ初期ロード時に旧バージョンのデータ（単一現場）が存在する場合は、新しい `projects` 配列の初回要素としてマイグレーションするロジックを実装しました。
- 画面ヘッダー部分に、作成済みの現場を切り替えるためのプルダウンメニューと、「新規追加」ボタン、「削除」機能を実装しました。
- 実績データの追加や削除、仕様項目の進捗率更新処理などを、常に `activeProject` (現在選択中の現場) に対して実行するようロジックを修正しました。

## テストと検証結果
開発サーバーを起動し、ブラウザエージェントによる以下の自動検証を実施しました。

1. **アクセスとマイグレーションの確認**
   - 既存データが正しく「岩渕淳子様邸外部塗替工事」として読み込まれ、全体の予測損益や進捗状況などの集計データが以前の状態を引き継いでいることを確認。
2. **新規現場の追加とデータ独立性の確認**
   - ヘッダーから新しい現場（「新規の現場」）を追加し、進捗や実績がすべて初期状態（0% / 0h）であることを確認。
   - 「工事設定」タブで現場名を「Test Project 2」へ変更。
3. **データの編集と永続化**
   - 「管理シート」タブで進捗率のスライダーを更新し、「実績入力」タブから日報を追加。
   - ブラウザをリロードしても、「Test Project 2」の名称や入力データが消去されずに保持されていることを確認。
4. **現場の切り替えテスト**
   - リロード後、最初にマイグレーションされた「岩渕淳子様邸」の現場へプルダウンから切り替え。
   - 新規作成した現場のデータと混在することなく、各現場固有の実績や進捗が正確に表示されることを確認。

以上のテストを通して、**複数現場の独立したデータ管理と永続化（切り替え機能）が正常に機能していること**を実証しました。
