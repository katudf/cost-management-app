# ウォークスルー: 仕入帳データのSupabase移行完了確認

## 移行内容の概要
1. **データベースのテーブル作成**:
   - Supabase 上に新しく `PurchaseRecords` テーブルを作成し、適切なデータ型を持たせました。
2. **初期データのアップロード完了**:
   - `docs/test_data/仕入帳.xlsx` 内の全データ（2125件）を解析し、作成したデータベースへ一括登録しました。
3. **フロントエンドの読み込み先変更**:
   - `PurchaseLedgerTab.jsx` で行っていたローカルファイルからのデータ読み込み処理 (`xlsx` 依存) を廃止しました。
   - 代わりに、Supabaseクライアントから直接 `PurchaseRecords` のデータを取得し、画面に表示するよう連携を完了しました。

## ユーザー様への確認手順
1. **アプリケーションの確認**:
   - 現在起動中の開発サーバー (`npm run dev`) をブラウザでお開きください。
2. **「仕入帳」タブの操作**:
   - アプリケーション上部の「仕入帳」タブをクリックします。
   - ローディング画面 ("データベースから仕入帳を読み込んでいます...") の後、以前と同様にテーブルのデータが表示されることを確認してください。
   - ※表示されているデータはローカルファイルからではなく、すべて Supabase データベースから取得されています。
3. **検索機能の確認**:
   - 右上の検索窓に任意のキーワード（例: 「サンリード」や名称の一部）を入力し、正しく絞り込まれることを確認してください。
