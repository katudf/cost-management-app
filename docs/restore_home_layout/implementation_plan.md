# ホームタブのレイアウト復元計画

ユーザーからの「ホームタブの登録済み工事一覧の配置が変わっている」という指摘に基づき、現在のカンバン（列）形式のレイアウトを、以前のグリッド（カード）形式および全体サマリー統計が表示されるレイアウトに復元します。

## 変更内容

### 1. `src/AdminApp.jsx` のホームタブ表示ロジックの修正
現在 `activeTab === 'dashboard'` のセクションに記述されているカンバン形式のコードを、以前のグリッド形式のコードに置き換えます。

- **全体サマリー統計の復元**: 「工事件数」「施工中の予測粗利合計」「施工中の総消化工数/目標」を表示する3つのカードを復活させます。
- **グリッドレイアウトの復元**: 工事一覧を 1列(モバイル) / 2列(タブレット) / 3列(PC) のカード形式で表示するように戻します。
- **検索機能の維持**: 現在導入されている検索バーは、グリッドレイアウトの上部に配置して維持します。
- **変数のマッピング**: `dashboardStats.displayProjects` や `dashboardStats.allProjectsSummary` などの最新のフックから提供される変数を使用して、整合性を保ちます。

## 検証計画

### 手動確認
- ホームタブ（「ホーム」ボタンをクリック）を表示し、上部にサマリー統計が表示されているか確認。
- その下に工事一覧がカード形式で並んでいるか確認。
- 検索バーで工事名を絞り込めるか確認。
- カードをクリックした際に、正しく「工事設定（マスター）」タブに遷移し、その工事が選択されるか確認。
