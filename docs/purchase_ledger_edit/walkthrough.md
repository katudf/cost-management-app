# ウォークスルー: 仕入帳データ編集機能の動作確認

## 変更内容の概要
1. **新規登録機能 (モーダルダイアログ)**:
   - 「仕入帳データ (DB)」ヘッダー部分に「新規登録」ボタンを追加しました。
   - ボタンを押下すると、画面中央にモーダルダイアログが表示され、新しい購入記録（月/日、工事名、購入先、名称など）を入力できます。
   - 「保存する」を押下すると、データベース(`PurchaseRecords`)へ新しい行が追加され、テーブルに即時反映されます。
2. **編集機能 (インライン編集)**:
   - 各データ行の「操作」列に「編集（ペンアイコン）」ボタンを追加しました。
   - 編集ボタンを押すと、その行のテキストが直接入力可能なフィールド（Input枠）に切り替わります（インライン編集モード）。
   - ここで内容を修正し、「保存（チェックアイコン）」を押下するとデータベースが更新されます。「キャンセル（バツアイコン）」を押すと変更が破棄されます。
   - ※ 単価や数量を変更した場合、金額が自動的に計算される簡易処理も組み込んでいます。
3. **削除機能**:
   - 各データ行の「操作」列に「削除（ゴミ箱アイコン）」ボタンを追加しました。
   - 押下時に確認ダイアログ（「本当にこのデータを削除しますか？」）が表示され、「OK」を押すとデータベースから該当データが永久に削除されます。

## 確認手順 (ユーザー様へのご案内)
1. **アプリケーションの準備**:
   - 稼働中の開発サーバー (`npm run dev`) をブラウザで開いていただき、「仕入帳」タブへと移動してください。
2. **新規登録のテスト**:
   - 「新規登録」ボタンを押し、ダミーの情報を入力して「保存する」ボタンを押してください。
   - テーブルの一番最後（または検索結果内）に、新しく追加したデータが表示されることを確認してください。
3. **編集のテスト**:
   - 既存の行または先ほど追加したダミー行の「編集」ボタン（ペンアイコン）を押します。
   - 数量や名称などの内容を書き換えて、「保存」ボタン（チェックアイコン）を押します。
   - 書き換えた内容がデータベースに保存され、一覧画面の表示が更新されることを確認してください。
4. **削除のテスト**:
   - 追加したダミー行の「削除」ボタン（ゴミ箱アイコン）を押します。
   - 確認ダイアログで「OK」をクリックし、その行が一覧から消える（データベースから削除される）ことを確認してください。
