# 実装計画: 仕入帳データ編集機能の追加

## 目的
仕入帳タブ (`PurchaseLedgerTab.jsx`) において、データベース上の `PurchaseRecords` テーブルに対する新規登録 (CREATE)、更新 (UPDATE)、削除 (DELETE) が行えるようにUIとロジックを拡張する。

## 提案する変更内容

### 1. 新規データの追加機能
#### [MODIFY] `src/components/tabs/PurchaseLedgerTab.jsx`
- 画面上部の検索窓の横に「新規登録」ボタン (`Plus` アイコン等) を配置します。
- クリック時に入力フォーム（モーダル、またはテーブルの先頭行への新規入力行追加）を表示し、必要なデータ（月/日、工事名、購入先、名称等の9項目）を入力可能にします。
- 保存時に `supabase.from('PurchaseRecords').insert()` を呼び出し、結果を画面に反映します。

### 2. 既存データの編集機能（インライン編集）
#### [MODIFY] `src/components/tabs/PurchaseLedgerTab.jsx`
- テーブルの右端に「操作 (Actions)」列を追加し、「編集 (`Edit3` アイコン)」ボタンを配置します。
- 編集ボタン押下で、対象行の各セルが `input` タグ（入力フィールド）に切り替わる「インライン編集モード」を実装します。
- 「保存 (`Check` アイコン)」ボタンで `supabase.from('PurchaseRecords').update().eq('id', row.id)` を呼び出しデータベースの値を更新します。
- 「キャンセル (`X` アイコン)」ボタンで編集状態を破棄して元の値を表示します。

### 3. データの削除機能
#### [MODIFY] `src/components/tabs/PurchaseLedgerTab.jsx`
- 「操作 (Actions)」列に「削除 (`Trash2` アイコン)」ボタンを配置します。
- クリック時、誤操作防止のためにブラウザ標準の確認ダイアログ (`window.confirm`) を表示します。
- 承認された場合、`supabase.from('PurchaseRecords').delete().eq('id', row.id)` を実行してデータを削除し、画面を更新します。
