# 作業員の並び順保存機能の修正（一意制約エラーの解消）

作業員一覧（作業員マスター設定）でドラッグ＆ドロップによる並べ替えを行った際、保存が失敗してリロードすると元に戻ってしまう問題を修正します。

## 現状と原因
- **エラー内容**: `409 Conflict` (PostgreSQL error `23505`: unique violation)
- **原因**: `Workers` テーブルの `display_order` カラムに `UNIQUE` 制約（一意性制約）が設定されているようです。並び替えで `1, 2, 3` と順番を振り直す際、一時的に既存のレコードの値と重複してしまい、更新が拒否されています。
- **副次的な問題**: 現在のコードでは `setWorkers`（ステート更新関数）の中で非同期の `Promise.all` を実行しており、React のルールに違反しているだけでなく、エラーハンドリングも不十分です。

## 修正内容

### 1. `useWorkers.js` の修正
並び替えロジックを以下のように変更します：
- 状態更新（Optimistic UI）とデータベースへの永続化を分離します。
- データベース更新時、一意制約エラーを回避するために、現在の `display_order` の最大値を取得し、そこから連番を振るようにします（この手法は本プロジェクト内の他箇所でも採用されています）。
- API呼び出しの結果を適切にチェックし、失敗時にはトーストを表示します。
