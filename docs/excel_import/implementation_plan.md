# 見積書Excel読み込み機能 (アプローチ3) 実装計画

ユーザーより提供されたテスト用Excelデータ（`●見積書（横-鏡あり）`）の構造を解析した結果、以下の仕組みでデータを抽出・取り込みできることが確認できました。

## 解析されたExcelデータの構造
見積内訳（シート名：`２内訳書`など）の行データは以下のように各列（カラム）に配置されています。
- **名称** (作業内容): 2列目 (配列のインデックス `2`)
- **仕様**: 24列目 (配列のインデックス `24`)
- **数量**: 40列目 (配列のインデックス `40`)
- **単位**: 43列目 (配列のインデックス `43`)

これらを利用して、有効な作業項目を抽出しアプリのマスターデータに変換します。

## UI・操作の設計
### [MODIFY] `src/App.jsx`
1. **「工事設定」タブ内のUI追加**
   - 「現場を削除」ボタンの横などに、「Excelファイルからインポート」のファイル選択ボタン（`<input type="file" accept=".xlsx, .xls" />`）を配置します。
2. **データの解析ロジック追加**
   - 選択されたファイルを `FileReader` で読み込み、`xlsx` ライブラリを使用してパースします。
   - `２内訳書` シート（または全シート）の行をループ処理します。
   - `名称 (row[2])` が存在し、かつ「合　計」などの小計行ではなく、`数量 (row[40])` が数値である行を「作業項目」として抽出します。
3. **マスターデータへの変換**
   - 抽出したデータを、`{ id: 連番, task: "名称 [仕様] (数量 単位)", target: 0 }` の形式に変換します。
     - 例：「　下塗(微弾性ﾌｨﾗｰ) [1回塗　淡彩色　はけ･ﾛｰﾗｰ] (236 m2)」
   - 抽出後、現在のマスターデータに追加する（または上書きする）機能を提供します（※今回は現在のデータを一旦上書きリセットしつつ取り込む方式とし、必要なら確認ダイアログを出します）。

## パッケージの追加
- Excelファイルをクライアント側（ブラウザ上）で直接解析するため、`xlsx` (SheetJS) パッケージをインストールします。

## 検証プラン
1. 開発サーバー (`npm run dev`) で起動。
2. 「工事設定」タブを開き、提供されたテスト用エクセルファイルを選択。
3. マスターデータ（仕様項目）が自動的に生成され、名称・仕様・数量が正しく連結して表示されることを確認。
4. 目標時間（手入力の部分）が `0` の状態で初期化されること、変更した際に保存されることを確認。
