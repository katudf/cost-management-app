# 見積金額表示機能 実装完了のご報告 (Walkthrough)

## 実装した機能の概要
管理シート（詳細画面）および工事設定タブにおいて、各作業項目の**見積金額**を保持し、表示・編集できる機能を追加しました。
また、ご要望の通り、見積金額の手動変更時に目標工数も自動的に再計算（見積金額 ÷ 4375円）されるように連動させています。

## 1. データベースの改修 (ユーザー作業)
Supabaseダッシュボード（SQLエディタ等）にて、以下のSQLを実行して `ProjectTasks` テーブルに新しいカラムを追加してください。
（このSQLは `docs/display_estimate_amount/schema.sql` にも保存しています）

```sql
-- 見積金額を保存するためのカラムを追加するSQL
ALTER TABLE public."ProjectTasks" 
ADD COLUMN estimated_amount numeric NOT NULL DEFAULT 0;
```

## 2. 変更・追加された機能詳細

### エクセル取り込み機能の強化
- エクセルファイル（見積書）をインポートする際、原本の「金額」列の数値を直接 `estimated_amount` としてデータベースに保存するよう対応しました。

### 工事設定タブ
- 各作業項目の列に新しく **「見積金額」の入力ボックス** を追加しました（目標時間入力欄の左横）。
- この「見積金額」の数値を変更してボックスからフォーカスを外すと、**自動的に目標時間が「見積金額 ÷ 4375h」で再計算**され、データベースと共に更新されます。
- ※ 今回の仕様に基づき、目標工数(h)側は自動計算項目となるため、直接の手動入力はできない状態（Read Only）に変更しています。

### 管理シートタブ（サマリー画面）
- 「項目別詳細予測」テーブルの一覧に、新しく **「見積金額」の列（表示専用）** を追加しました。
- これにより、各タスクのもともとの見積額と、予測粗利や消化状況を簡単に比較できるようになりました。

### エクセル出力機能の更新
- 前回のタスクで追加された「Excel出力」機能におきましても、同様に「作業項目別詳細」のシート内に **「見積金額(円)」** 列を出力するように改修しています。

## 動作確認の手順
1. **DBの反映**: SupabaseのSQLエディタで上記のSQLを実行し、カラムを追加してください。
2. **インポート確認**: 管理アプリの「工事設定」タブから新しいエクセルを読み込みます。
3. **工事設定タブの確認**:
   - 読み込まれた各タスクに「見積金額」が表示されていることを確認します。
   - 金額の数値を手動で変更し、右側の「目標時間」が自動的に変わることを確認します。
4. **管理シートの確認**: 「管理シート」タブを開き、下部のテーブルに見積金額列が追加され、正しく表示されているか確認します。
5. **出力確認**: 「Excel出力」ボタンを押し、ダウンロードしたエクセルファイル内の項目別シートに見積金額が出力されていることを確認します。
