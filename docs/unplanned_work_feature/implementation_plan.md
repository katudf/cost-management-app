# 見積外作業の対応方針（案2）実装計画

ユーザーからの要望に基づき、見積もりにない追加作業が発生した場合の運用を「案2（作業員が任意で新規作業項目を追加できる運用）」にて実装するための計画です。

また、前回の実装で追加したAdminApp.jsxでの「その他作業」自動追加ロジックは撤回し、元の状態に戻します。（ただしバグ修正で入れたorderキーの修正のみ保持します）

## 提案する変更内容

### 1. `src/AdminApp.jsx` のロールバック

#### [MODIFY] `src/AdminApp.jsx`
*   **現場新規作成時の自動追加の撤回:**
    `addNewProject` 関数にて挿入した「その他作業」の自動追加処理を削除します。
*   **Excelインポート時の自動追加の撤回:**
    `handleImportChoice` 関数にて挿入した末尾への「その他作業」追加処理を削除します。

### 2. `src/WorkerApp.jsx` の修正

#### [MODIFY] `src/WorkerApp.jsx`
*   **「新しい作業項目を追加」ボタンの設置:**
    作業実績を入力するメイン画面の作業項目リストの一番下に、「新しい作業項目を追加」ボタン（破線枠等の目立つデザイン）を配置します。
*   **追加処理の実装 (`handleAddNewTask`):**
    *   ボタン押下時、`window.prompt`等で追加したい作業の名称をユーザーに入力させます。
    *   入力された名称で `ProjectTasks` に新しいタスクを `insert` します（`target_hours: 0`, `estimated_amount: 0`）。
    *   `insert` 成功後、ローカルの `tasks` ステートに新しいタスクを追加し、即座にそのタスクに対する作業時間入力ができるようにします。
*   **アイコンのインポート追加:**
    `lucide-react` から `PlusCircle` アイコンをインポートし、ボタンに使用します。

## 検証計画
1.  **Admin側の確認:**
    *   現場の新規作成やExcelインポートを行った際、「その他作業」が勝手に追加されないことを確認する。
2.  **Worker側の確認:**
    *   WorkerAppにログインし、対象の現場を開く。
    *   作業項目リストの下部に「新しい作業項目を追加」ボタンが表示されていることを確認する。
    *   ボタンを押し、任意の作業名（例：「追加資材の運搬」）を入力して追加できることを確認する。
    *   追加された項目に対して時間実績を入力でき、保存できることを確認する。
