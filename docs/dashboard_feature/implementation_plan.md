# 実装計画: ダッシュボード（工事一覧）画面の追加

## 概要
現在、アプリのトップレベルには特定の現場の「管理シート」「実績入力」「工事設定」が表示されていますが、全体を俯瞰できる「ホーム（ダッシュボード）」画面が存在していません。
本機能では、登録されているすべての現場（工事）を一覧表示し、それぞれの全体の進捗・工数・粗利状況をカード形式で視覚的に確認できるダッシュボードを新設します。

## 変更内容

### 1. `src/App.jsx`
- **ステートの追加・変更**
  - `activeTab` の初期値を `'summary'` から `'dashboard'` に変更し、最初の画面をダッシュボードにします。
  - ナビゲーションメニューに「ホーム」タブを追加します。
- **全現場データの集計ロジック (`allProjectsSummary`)**
  - 現在の個別プロジェクト用の集計ロジック（`summaryData`）を参考に、全プロジェクト（`projects`）をループして、現場ごとの以下情報を算出する用メモ関数（`useMemo`）を作成します。
    - `totalActual` (合計実績工数)
    - `totalTarget` (合計目標工数)
    - `totalPredictedProfitLoss` (予測粗利合計)
- **ダッシュボードUIの追加**
  - `activeTab === 'dashboard'` の際のレンダリングブロックを追加します。
  - グリッドレイアウト（`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`）を使用し、算出した各現場のデータをカード状に表示します。
  - 現場カードに「詳細を見る」用のボタン／またはカード全体のクリックイベントを配置し、クリック時に `setActiveProjectId(project.id)` および `setActiveTab('summary')` を発行して現場詳細画面に遷移させます。

## 検証計画
- `npm run dev` でローカルサーバーを起動し確認。
- アプリを開いた際、デフォルトで「ホーム」タブが開き、複数の現場カードが表示されているか（現場がなければ「現場がありません」等のメッセージ）。
- カードから特定の現場をクリックした際、正しくその現場が選択された状態で「管理シート」に切り替わるか検証する。
