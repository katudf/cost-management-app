# ダッシュボード機能追加のタスクリスト

- [ ] 1. 事前準備と設計
  - [ ] 1.1 `App.jsx` 内のタブ管理ステート（`activeTab`）の初期値を `dashboard` に変更する。
  - [ ] 1.2 全プロジェクトのデータを集計するためのヘルパー関数または `useMemo` を作成する（各現場の目標合計、実績合計、全体進捗率などを算出）。
- [ ] 2. UIの構築
  - [ ] 2.1 ナビゲーションメニューの先頭に「ホーム」または「一覧」ボタンを追加し、タブを切り替えられるようにする。
  - [ ] 2.2 `activeTab === 'dashboard'` の際の表示エリア（メイン画面）を作成する。
  - [ ] 2.3 登録済み工事の一覧をカード形式（グリッド表示）で表示するデザインを構成する。
- [ ] 3. 機能の実装
  - [ ] 3.1 各現場カード（プロジェクトカード）に、以下の情報を表示させる。
    - 現場名
    - 消化済工数 / 全体目標工数
    - 予測粗利のステータス（カラーなどで黒字/赤字を視覚化）
  - [ ] 3.2 現場カードをクリック（または「詳細を見る」ボタンを押す）で、その現場を選択状態（`activeProjectId`）に変更し、「管理シート（summary）」タブへ遷移する処理を実装する。
- [ ] 4. テストと動作確認
  - [ ] 4.1 複数の現場を作成し、ホーム画面で一覧表示されるか確認する。
  - [ ] 4.2 各現場の集計値が正しく算出・表示されているか確認する。
  - [ ] 4.3 現場カードから詳細画面へのスムーズな遷移が行えるかテストする。
