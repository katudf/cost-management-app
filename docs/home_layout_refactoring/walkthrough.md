# ホーム画面リファクタリング：完了報告（同ステータス内並び替えバグ修正版）

同ステータス（カラム）内でのドラッグによる並び替えが正常に画面に反映されなかったバグの修正を完了しました。

---

## バグの原因と修正内容

### 【原因】
1. **クライアント側での強制並び替え**:
   * ダッシュボード表示用のプロジェクトリストを生成するカスタムフック `useDashboardStats.js` 内の `displayProjects` メソッドにおいて、表示順序が常に `b.id - a.id` (IDの降順、作成が新しい順) で強制的にソートし直されていました。
   * これにより、DB側やフック内でどれだけ `order` を正しく書き換えても、画面上では常に作成日時の新しい順に戻ってしまっていました。
2. **プロジェクト取得クエリのソート基準**:
   * `useSupabaseData.js` 内の初期ロード・再フェッチ用クエリが `created_at` 順でソートされており、`order` カラムの値がロード順に反映されていませんでした。

### 【対策】
1. **表示ソートロジックの修正 (`useDashboardStats.js`)**:
   * `displayProjects` のソート条件を、`order` カラムの昇順（値の小さいものが上）になるように変更しました。
   * `order` カラムが設定されていない、または同値の場合は、フォールバックとしてIDの降順（新しい順）でソートします。
2. **フェッチ順の変更 (`useSupabaseData.js`)**:
   * プロジェクトリストのロード時、Supabase から `order` カラムの昇順でソートして取得するようにクエリを変更しました。
