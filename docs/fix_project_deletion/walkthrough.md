# 現場削除時のエラー修正 (Walkthrough)

## 修正の概要
「工事設定」タブから現場（プロジェクト）自体を削除する際に、データベースの外部キー制約エラーによって生じていた不具合を解消しました。

## エラーの原因
`Projects` テーブルを削除する際、その現場に紐づいている `ProjectTasks` や `TaskRecords` などのレコードがデータベース上に残っていると、安全のために作動する「外部キー制約（参照整合性）」によって削除処理がブロック（Conflictエラー）されていました。

## 対応内容
- `AdminApp.jsx` 内にある `removeProject` 関数を修正しました。
- `Projects` テーブルから対象の現場を削除する直前に、以下の関連データを**順番に削除する処理（クリーンアップ）**を追加しました。
  1. `SubcontractorRecords` (該当現場の協力業者実績)
  2. `TaskRecords` (該当現場の日報・実績データ)
  3. `ProjectTasks` (該当現場の作業項目・マスターデータ)
  4. 最終的に `Projects` 本体の削除を実行

これによって、エラーなく対象の現場をシステム上から完全に削除できるようになりました。
