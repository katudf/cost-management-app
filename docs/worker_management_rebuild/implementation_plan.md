# 作業員管理タブ再構築 計画書

「作業員」管理タブを、新たに追加された `Workers` テーブルの複数カラム（生年月日、入社日、住所、連絡先、CPDS番号、ふりがな等）に対応できるよう再構築します。

## 背景と目的
現在の作業員管理UIは名前の変更と並び順の入れ替えしかできないシンプルなリストですが、今後は住所や電話番号などの詳細なプロフィール情報も管理・表示できるようにします。

## Proposed Changes

### [AdminApp.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/AdminApp.jsx)

#### 1. データ取得処理の更新
*   `fetchAllData` 関数内の `Workers` データ取得クエリを更新します。
    *   現状: `.select('id, name, birthDate')`
    *   変更後: `.select('*')` (もしくは全カラムを明示指定) とし、すべてのプロフィール情報をフロントエンドで利用できるようにします。

#### 2. 新規 State の追加
*   詳細情報を編集・新規作成するためのモーダル用Stateを追加します。
    *   `isWorkerModalOpen` (boolean): モーダルの開閉状態
    *   `editingWorker` (object | null): 編集対象の作業員データ。新規作成時は空の構造体、編集時は既存のデータが入ります。

#### 3. 「作業員」タブ UIの再構築
*   左側の「作業員マスター設定」リストを、より情報量を含められるリスト表示（またはテーブル表示）に変更します。
    *   表示項目例: 名前、ふりがな、連絡先、年齢（生年月日から計算）、[編集] ボタン、[削除] ボタン
    *   順序の入れ替え（↑↓矢印）は維持し、スマホアプリ等の表示順序コントロール機能をそのまま残します。
    *   既存の `prompt` を使った「追加する」ボタンの動作を、新規作成モーダルを開く動作に変更します。

#### 4. 詳細編集・追加モーダルの実装
*   名前、ふりがな、生年月日(date)、入社日(date)、住所、連絡先、CPDS番号を入力できるフォームを持ったモーダルを作成します。
*   「保存」ボタン押下時に、新規の場合は `supabase.from('Workers').insert()` を、既存の編集の場合は `update()` を実行するロジックを実装します。
