# 実装計画: 作業員向けスマホ入力アプリ

## 概要
現在事務員や管理者が使っているPC向け画面（詳細工数管理システム）とは完全に分けた、作業員が現場からスマホで簡単にアクセスできる日報入力画面（Worker App）を構築します。同じリポジトリと同じSupabase DBを共有します。

## アーキテクチャと画面設計（ルーティング）
今回、外部ルーティングライブラリ（`react-router-dom`など）は新たに追加せず、現在のVercelのSPA構成を維持し、シンプルなコンポーネントの切り替えによる疑似ルーティング（またはURLパラメータ判定）を採用します。

- アクセスURLの想定：
  - `https://[アプリURL]/` または `https://[アプリURL]/?mode=admin` → 現在のPC向け管理画面
  - `https://[アプリURL]/?mode=worker` → 今回開発する作業員向けスマホアプリ画面

## 必要なデータベース改修 (Supabase)
誰が「職長」であるかを判定するため、`Projects`（現場）テーブルに担当職長のIDを保存するカラムを追加します。
- `Projects` テーブルに `foreman_worker_id` (bigint, nullable) カラムを追加。
- 外部キーとして `Workers` テーブルの `id` に紐づく想定ですが、クライアント側だけでも制御可能です。（今回はシンプルに `foreman_worker_id` という数値を保存します）

## 管理画面（Admin App）への変更点
職長を指定できるように、`src/App.jsx`（現 `AdminApp` 相当）の **「工事設定」** 画面に改修を加えます。
1. 各「管理現場名」の入力欄の下などに、「担当職長」を選択するプルダウンを追加します（選択肢は `Workers` テーブルから取得した作業員リスト）。
2. これを `Projects.foreman_worker_id` に保存・更新できるようにします。

## 作業員アプリ（Worker App）の画面フロー
新しいコンポーネント `src/WorkerApp.jsx` を作成します。

### 1. ログイン画面
- **機能**: `Workers` テーブルから取得した作業員名が大きなボタン、または分かりやすいプルダウンで並んでいます。
- **操作**: 自分の名前を選んで「ログイン」ボタンを押すだけ（パスワード不要）。
- **状態保持**: ログインした `workerId` と `workerName` をブラウザの `localStorage`（例:`cost-app-worker`）に保存し、次回アクセス時はこの画面をスキップして自動ログインします。

### 2. メイン画面（日報入力）
- **UI要件**: スマホでタップしやすい大きなフォントと余白。
- ヘッダーに「ログイン中の名前」と「ログアウト」ボタン。
- **操作ステップ**:
  1. **現場の選択**: 現在進行中の現場（Projects）から、今日入った現場を選択します。（プルダウンまたはリスト）
  2. **作業時間の入力**:
     - 選んだ現場の `ProjectTasks`（作業項目）がリスト表示されます。
     - 各項目に `[ - ] [ 0 ] [ + ]` のようなボタン、または数値入力枠があり、今日使った「時間」を入力します。
     - 一番下の「日報を送信」ボタンを押すと、`TaskRecords` テーブルへインサート（または本日の自分のレコードをアップデート）します。
  3. **進捗率の入力（職長のみ）**:
     - 選択した現場の `foreman_worker_id` と、現在ログイン中の自分の `workerId` が一致した場合のみ、各作業項目の下に「現在の進捗率」をスライダーやボタンで変更できる専用のUIが表示されます。
     - （一般作業員にはこのUIは一切表示されません）
     - 送信ボタンを押した際、実績とともに `ProjectTasks.progress_percentage` を更新します。

## 実装ステップの提案
1. `src/main.jsx` で URLのクエリパラメータ (`?mode=worker`) を判定し、`<App />` を表示するか `<WorkerApp />` を表示するか分岐するロジックを実装。
2. `src/WorkerApp.jsx` を完全新規で作成し、モックアップUIを実装。
3. `src/App.jsx`（既存コード）で、現場ごとに職長を設定できるUIとロジックを追加（SupabaseへのALTER TABLEも行う）。
4. `WorkerApp.jsx` にSupabase連携を組み込み、ログイン保持・実績入力・職長進捗入力を完成させる。
