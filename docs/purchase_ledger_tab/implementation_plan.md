# 実装計画: 仕入帳タブの追加

## 目的

システムに `docs/test_data/仕入帳.xlsx` の内容を表示する専用タブを追加し、ブラウザ上で仕入データを閲覧・検索できるようにする。

## 提案する変更内容

### 1. 仕入帳タブコンポーネントの新規作成

#### [NEW] `src/components/tabs/PurchaseLedgerTab.jsx`

- `xlsx` ライブラリを利用し、Vite のアセットURL (`?url`) インポート機能を用いて `docs/test_data/仕入帳.xlsx` を読み込む処理を実装します。
- `fetch` を使用して取得したデータを ArrayBuffer として読み込み、`xlsx.read` で解析します。
- 読み込んだデータをヘッダー（1行目）とデータ行（2行目以降）に分割し、Stateとして保持します。
- Tailwind CSS を活用したレスポンシブなテーブルデザインにし、スクロール可能な領域にデータを表示します。
- 利便性向上のため、工事名や名称、購入先などで絞り込める検索窓を配置します。

### 2. メインアプリケーションの変更

#### [MODIFY] `src/AdminApp.jsx`

- 画面上部のナビゲーションタブ一覧に `purchase_ledger`（仕入帳）を追加し、ボタンを押下できるようにします。
- `import PurchaseLedgerTab from './components/tabs/PurchaseLedgerTab';` を追加します。
- `activeTab === 'purchase_ledger'` の場合に `<PurchaseLedgerTab />` をレンダリングするように条件分岐を追加します。

## 検証計画 (Verification Plan)

- **手動検証**: ブラウザ (`npm run dev` で起動中の開発サーバー) でアプリケーションを開き、以下の点を確認します。
  1. ナビゲーションに「仕入帳」タブが表示され、クリックで切り替わること。
  2. 仕入帳タブを開いた際、`docs/test_data/仕入帳.xlsx` の内容がテーブルに正しく表示されること。
  3. テーブルのヘッダー（月/日, 工事名, 購入先, 名称, 数量, 単価, 金額など）が列と一致していること。
  4. 検索ボックスに入力したキーワードで、行が正しくフィルタリングされること。
