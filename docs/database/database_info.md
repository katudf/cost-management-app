# データベース定義書

本システムで使用されている Supabase (PostgreSQL) のテーブル定義およびスキーマ情報です。

---

## 1. プロジェクト・工事関連

### Projects (プロジェクト)
現場全体の基本情報を管理します。

```sql
CREATE TABLE public.Projects (
  id int8 NOT NULL PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  name text,                -- 現場名
  order int8 NOT NULL,       -- 表示順（旧）
  display_order int8,        -- 表示順（配置図用）
  startDate date,            -- 工期 開始日
  endDate date,              -- 工期 終了日
  estimatedAmount int8,      -- 見積総額
  finalAmount int8,          -- 最終金額
  taxAmount int8,            -- 消費税額
  isPaid bool,               -- 支払完了フラグ
  estimatePdfUrl text,       -- 見積書PDF
  contractPdfUrl text,       -- 契約書PDF
  status text,               -- ステータス（見積/予定/施工中/完了）
  customerId int8,           -- 顧客ID
  bar_color text,            -- 配置図での表示色
  foreman_worker_id int8,    -- 職長（作業員ID）
  is_prime_contractor bool DEFAULT false -- 元請フラグ
);
```

### ProjectTasks (作業項目・仕様)
現場内の具体的な作業項目（見積明細）と目標工数を管理します。

```sql
CREATE TABLE public.ProjectTasks (
  id int8 NOT NULL PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  projectId int8,            -- 関連プロジェクトID
  name text,                 -- 作業項目名
  estimated_amount numeric NOT NULL, -- 見積金額
  target_hours int8,         -- 目標工数（h）
  progress_percentage int8,   -- 進捗率（0-100）
  order int8,                -- 表示順
  startDate date,            -- 開始予定
  endDate date,              -- 終了予定
  status text,               -- 状態
  serviceMasterId int8       -- マスターID（紐付け用）
);
```

### ProjectSuspensions (休工期間)
工事期間中の非稼働期間を管理します。

```sql
CREATE TABLE public.ProjectSuspensions (
  id int8 NOT NULL GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at timestamptz NOT NULL DEFAULT now(),
  project_id int8 NOT NULL,  -- 関連プロジェクトID
  start_date date NOT NULL,   -- 休工開始日
  end_date date NOT NULL,     -- 休工終了日
  reason text                -- 理由（例：他現場対応）
);
```

## 3. 見積関連

### estimates (見積ヘッダー)
作成された見積書の基本情報を管理します。

```sql
CREATE TABLE public.estimates (
  id              int8        GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now(),
  deleted_at      timestamptz,
  estimate_number text        UNIQUE NOT NULL,   -- 見積番号
  customer_id     int8        NOT NULL REFERENCES public."Customers"(id),
  project_id      int8        REFERENCES public."Projects"(id),
  created_by      int8        REFERENCES public."Workers"(id),
  title           text        NOT NULL,          -- 件名
  site_location   text,                          -- 現場住所
  work_period     text,                          -- 工期（テキスト）
  issue_date      date        NOT NULL DEFAULT CURRENT_DATE, -- 発行日
  valid_until     date,                          -- 有効期限
  payment_terms   text        NOT NULL DEFAULT '従来通り',   -- 支払条件
  notes           text,                          -- 備考
  status          text        NOT NULL DEFAULT 'draft' CHECK (status IN ('draft','submitted','accepted','rejected')),
  tax_rate        numeric     NOT NULL DEFAULT 0.10,
  show_fixed_fees bool        NOT NULL DEFAULT true,
  show_net        bool        NOT NULL DEFAULT true,
  show_subtotals  bool        NOT NULL DEFAULT false,
  stamp_header    text        NOT NULL DEFAULT 'company' CHECK (stamp_header IN ('company','representative','none')),
  show_approver   bool        NOT NULL DEFAULT false
);
```

### estimate_items (見積明細)
見積書内の各作業項目を管理します。階層構造（親子関係）に対応しています。

```sql
CREATE TABLE public.estimate_items (
  id              int8        GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at      timestamptz NOT NULL DEFAULT now(),
  estimate_id     int8        NOT NULL REFERENCES public.estimates(id) ON DELETE CASCADE,
  parent_id       int8        REFERENCES public.estimate_items(id) ON DELETE CASCADE,
  sort_order      int4        NOT NULL DEFAULT 0,
  item_type       text        NOT NULL CHECK (item_type IN ('category','item','subtotal','fixed')),
  category_symbol text,                          -- カテゴリ記号（A, B...）
  name            text        NOT NULL,          -- 項目名
  spec            text,                          -- 仕様/摘要
  quantity        numeric,                       -- 数量
  unit            text,                          -- 単位
  unit_price      numeric,                       -- 単価
  amount          numeric,                       -- 金額
  note            text                           -- 備考
);
```

## 5. ビュー・関数・トリガー

### v_estimate_category_totals (見積カテゴリ別集計ビュー)
見積書内のカテゴリごとの合計金額を算出するビューです。

```sql
CREATE VIEW public.v_estimate_category_totals AS
SELECT
  e.id                          AS estimate_id,
  e.estimate_number,
  e.title                       AS estimate_title,
  i.id                          AS category_item_id,
  i.category_symbol,
  i.name                        AS category_name,
  SUM(CASE WHEN c.item_type = 'item' THEN COALESCE(c.amount, 0) ELSE 0 END)
                                AS category_amount
FROM public.estimates e
JOIN public.estimate_items i ON i.estimate_id = e.id AND i.item_type = 'category'
LEFT JOIN public.estimate_items c ON c.parent_id = i.id AND c.item_type = 'item'
WHERE e.deleted_at IS NULL
GROUP BY e.id, e.estimate_number, e.title, i.id, i.category_symbol, i.name;
```

### 自動計算ロジック
- **金額自動計算**: `estimate_items` の挿入・更新時、数量と単価から金額を自動計算します。
- **更新日時管理**: `estimates` テーブルのレコード更新時、`updated_at` を自動的に現在日時に更新します。
- **見積番号採番**: 日付ベースの接頭辞に基づき、次なる見積番号（連番4桁）を取得する関数 `get_next_estimate_seq` を使用します。

## 7. セキュリティ (RLS)

本システムは未ログイン状態（`anon` ロール）でも運用可能なように、以下の通り RLS ポリシーを設定しています。

### estimates / estimate_items
見積書機能に関連するテーブルです。

```sql
-- 既存の制限の厳しいポリシーを削除
DROP POLICY IF EXISTS "estimates_all"      ON public.estimates;
DROP POLICY IF EXISTS "estimate_items_all" ON public.estimate_items;

-- 未ログインユーザー(anon)およびログイン済みユーザー(authenticated)の両方に権限を付与
CREATE POLICY "estimates_all" ON public.estimates
  FOR ALL TO anon, authenticated USING (deleted_at IS NULL);

CREATE POLICY "estimate_items_all" ON public.estimate_items
  FOR ALL TO anon, authenticated USING (true);
```
