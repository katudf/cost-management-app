# 未設定プロジェクトの工期クイック設定機能の追加計画

配置表（ガントチャート）において、工期（開始日・終了日）が未設定の工事についても表示し、カレンダー上のセルをクリックすることで即座に工期を設定できる機能を追加します。

## ユーザーレビューが必要な事項

- **表示対象**: 工期未設定の工事も配置表に表示されるようになります。これにより、工事リストが長くなる可能性がありますが、現在の「予定」「施工中」ステータスの工事をすべて表示する方針で進めます。
- **設定内容**: 空白セルをクリックした際、その日を開始日および終了日（1日間の工期）として初期設定します。その後はバーの端をドラッグして期間を調整可能です。

## 提案される変更内容

### [AssignmentChartTab.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/tabs/AssignmentChartTab.jsx)

- **データ取得ロジックの修正 (`fetchAssignments`)**: 
  - `Projects` テーブルの取得クエリから `startDate` / `endDate` が null でないことという条件を削除し、工期未設定の工事も取得するようにします。
- **バー計算ロジックの修正 (`getBarSpan`)**:
  - `startDate` または `endDate` が null の場合に `null` を返し、エラーを防ぐガード節を追加します。
- **工期設定ハンドラの追加**:
  - `handleProjectCellClick` を実装し、クリックされた工事と日付を元に Supabase とローカルステートを更新します。
- **UI（レンダリング）の修正**:
  - プロジェクト行の各セル (`td`) に `onClick` ハンドラを追加します。
  - 工期未設定のプロジェクトの場合のみ、クリックで工期設定が行われるように制御します。

## 確認計画

### 手動確認事項
1. 「工事一覧」などで、工期が未設定の新しい工事（ステータス：予定）を作成します。
2. 配置表タブを開き、その工事名がリストにあることを確認します（バーはまだ表示されていません）。
3. その工事の行の任意の日付セルをクリックします。
4. クリックした日に1日分のバーが表示され、トースト通知で「工期を設定しました」と表示されることを確認します。
5. ページをリロードしても、設定した工期が維持されていることを確認します。
