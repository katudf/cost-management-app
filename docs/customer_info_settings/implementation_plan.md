# 休日カレンダーの年度表示（4月〜翌年3月）への変更計画

設定タブ内の会社休日カレンダーを、現在の1月始まり（暦年）から、日本のビジネスシーンで一般的な4月始まり（年度）の表示に変更します。

## ユーザーレビューが必要な事項

- **年度の定義**: カレンダーの表示を「2024年」から「2024年度」とし、2024年4月〜2025年3月を表示するようにします。
- **初期表示**: 現在の日付に基づき、現在の年度（例：現在が2024年4月〜2025年3月の間であれば2024年度）を初期状態で表示します。

## 提案される変更内容

### [HolidayCalendar.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/HolidayCalendar.jsx)

- **状態の初期化**: `year` ステートを現在の「年度（Fiscal Year）」で初期化するように修正します。
- **データ取得ロジックの修正**: `fetchHolidays` で取得する日付範囲を、選択された年度の4月1日から翌年3月31日までに変更します。
- **レンダリングロジックの修正**: 
  - `renderMonth` 関数で、月番号が11（12月）を超えた場合に翌年の日付として処理するように修正します（`dateStr` の生成や休日カウントなど）。
  - 月のループを 0〜11（1月〜12月）から 3〜14（4月〜翌年3月）に変更します。
- **表示テキストの修正**: ヘッダーの「年」を「年度」に変更し、年度管理であることを明確にします。

## 確認計画

### 手動確認事項
1. 設定タブの「休日設定」を開き、カレンダーが4月から始まっていることを確認。
2. 4月から翌年3月までの12ヶ月分が表示されていることを確認。
3. 2025年1月〜3月のセルをクリックして休日を設定し、正しく保存・反映されることを確認。
4. 前年度・翌年度への切り替えが正しく動作することを確認。
5. 配置表側で、設定した休日が正しくグレーアウト（または赤表示）されることを確認。
