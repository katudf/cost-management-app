# 休日カレンダーの年度表示への変更 - 修正内容の確認

会社休日カレンダーを、4月始まりの「年度単位（4月〜翌年3月）」での表示・管理が可能な形式に変更しました。

## 実施した変更内容

### 1. カレンダーの表示期間・順序の変更
- **[HolidayCalendar.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/HolidayCalendar.jsx)**
    - 表示の始まりを「1月」から「4月」に変更しました。
    - 4月から翌年の3月までの12ヶ月分が一度に表示されるようになり、日本の年度区切りに合わせた休日管理が可能です。

### 2. 年度ベースのデータ管理ロジック
- **[HolidayCalendar.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/HolidayCalendar.jsx)**
    - **初期表示**: 現在の日付から現在の年度を自動計算して表示します（例：1月〜3月の間は前年の年度を表示）。
    - **データ取得範囲**: 選択された年度の「4月1日〜翌年3月31日」を対象に休日データを取得するように修正しました。
    - **年跨ぎの処理**: レンダリング時に1月〜3月分を自動的に「翌年」の日付として処理し、DBへの保存や読み込みが正しく行われるように実装しました。

### 3. UI・表記の改善
- ヘッダーの表記を「2024年」から「2024年度」に変更しました。
- 年跨ぎが発生する月（1月〜3月）でも、正しい月名と日付、休日数が表示されるように調整しました。
