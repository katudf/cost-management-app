# 配置表バーのドラッグ操作不具合の修正 - 修正内容の確認

カラー変更機能を導入した影響で、配置表バーのドラッグによる工期の移動や伸縮ができなくなっていた不具合を修正しました。

## 実施した変更内容

### イベント伝播の整理（ドラッグとクリックの両立）
- **[AssignmentChartTab.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/tabs/AssignmentChartTab.jsx)**
    - カラー選択パレットを表示するトリガーを、セルの `onClick` から `handlePointerUp`（マウスボタンを離した時）に変更しました。
    - 具体的には、マウスのドラッグ開始時（`pointerDown`）の座標と終了時（`pointerUp`）の座標を比較し、「**ドラッグによる移動が行われなかった場合のみクリックとみなす**」という判定を実装しました。
    - これにより、バーをドラッグで移動・伸縮させる操作と、クリックでカラーを変更する操作がシステム上で明確に区別して処理されるようになり、両方の機能が完全に共存できるようになりました。

## 確認事項

- [x] 配置表バーの両端をドラッグすると、工期が伸縮できる。
- [x] 配置表バーの中央部分をドラッグすると、工期全体が前後に移動できる。
- [x] 配置表バーをドラッグして移動中にマウスボタンを離しても、カラーパレットは開かない。
- [x] 配置表バーをクリック（ドラッグせずその場で離す）すると、カラーパレットが開く。

> [!TIP]
> クリックとドラッグの判定が厳密になり、直感的な操作が阻害されなくなりました。
