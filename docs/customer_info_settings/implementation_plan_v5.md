# 配置表カラー管理の刷新：ガントチャート上での直接変更機能の実装計画

配置表（ガントチャート）のバーを直接クリックしてカラーを変更できるようにし、工事設定画面の重複するカラー設定項目を削除します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> - 配置表のバーをクリックすると、その場でカラーパレット（DEFAULT_COLORS）が表示されるようにします。
> - 色を選択すると即座にデータベース（`Projects` テーブルの `bar_color`）が更新され、他の画面にも反映されます。
> - 工事設定タブ（`MasterTab.jsx`）からは、従来のカラー選択 UI を削除します。

## 提案される変更点

### 1. 配置表でのカラー変更機能の実装 [AssignmentChartTab]

#### [MODIFY] [AssignmentChartTab.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/tabs/AssignmentChartTab.jsx)
- **ステートの追加**: カラーパレットの表示状態（どのプロジェクトか、表示位置など）を管理する `editColorPopup` ステートを追加します。
- **クリックハンドラの追加**: ガントチャートのバー（`barProjects.map` 内の `div`）に `onClick` ハンドラを追加し、ポップアップを表示するようにします。
- **ポップアップ UI の実装**: `DEFAULT_COLORS` を使用した円形ボタンのパレットを表示するポップアップを実装します。 `editCell` ポップアップと同様の浮遊ボックス形式とします。
- **保存ロジックの実装**: 色が選択された際に、`supabase.from('Projects').update({ bar_color: color })` を実行し、同時にローカルの `projects` / `barProjects` ステートを更新します。

### 2. 工事設定画面の UI 整理 [MasterTab]

#### [MODIFY] [MasterTab.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/tabs/MasterTab.jsx)
- 「配置表カラー」セクション（ラベル、カラー選択ボタン群）を完全に削除します。
- これにより、工事基本設定画面がよりシンプルになります。

## 修正内容の確認方法

### 手動確認
1. **配置表での操作**: 配置表タブを開き、任意の案件バーをクリックします。
2. **カラーパレットの表示**: クリックした位置付近にカラーパレットが表示されることを確認します。
3. **色の変更**: パレットから別の色を選択し、バーの色が即座に変わることを確認します。
4. **永続化の確認**: 画面をリロードしても、変更した色が維持されていることを確認します。
5. **設定画面の確認**: 工事設定タブを開き、「配置表カラー」の項目が消えていることを確認します。
