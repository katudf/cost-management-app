# 土曜日の稼働日設定変更 - 修正内容の確認

土曜日の扱いを、会社休日が設定されていない限り「平日（稼働日）」として扱うように変更しました。

## 実施した変更内容

### 1. 休日判定ロジックの刷新
- **[AssignmentChartTab.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/components/tabs/AssignmentChartTab.jsx)**
    - `isHoliday` 関数および、レンダリング時の `isHolidayOrWeekend` 判定を修正しました。
    - **日曜**: 以前と同様、常に休日として扱われます。
    - **土曜**: 「会社休日」として特定の日付が登録されている場合のみ休日とし、それ以外は平日（バーの色が付く稼働日）として扱います。
    - **平日**: 「会社休日」として登録されている場合のみ休日となります。

### 2. カレンダーヘッダー表示
- 土曜日のヘッダー表示（青色）は維持しました。これにより、一目で土曜日であることを判別しつつ、バーが表示されていることで「稼働日であること」を同時に示します。
- 一方、土曜日を「休日」として登録した場合は、日曜と同様の赤色の表示に切り替わります。

## 確認事項

- [x] 会社休日が設定されていない土曜日にも、プロジェクトバーの色が表示されている。
- [x] 土曜日を独自に「休日」として登録した場合、その日のバー表示は中断され、背景が休日色（赤系）になる。
- [x] 日曜日は引き続き、常に休日として扱われ、バーの色は表示されない。

> [!NOTE]
> 「土曜日は基本的に稼働するが、特定の土曜だけ休む」という運用に最適化されました。
