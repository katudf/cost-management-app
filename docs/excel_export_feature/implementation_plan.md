# エクセル出力機能 実装計画

## 背景・目的
各現場の利益（予測粗利）、全体の延べ人数、作業項目ごとの延べ人数をまとめた帳票をエクセル形式で出力したいという要望に対応します。
すでに `xlsx` ライブラリがプロジェクトに導入されているため、これを活用してブラウザ上で直接エクセルファイルを生成・ダウンロードする機能を実装します。

## 出力するエクセル帳票の仕様案
出力するエクセルファイルには、主に以下の2つのシートを含めることを提案します。

### シート1: 現場別サマリー (全現場の総括)
- **対象**: 選択された現場
- **出力項目**:
  - 現場名
  - 予測粗利 (全体の利益)
  - 自社作業員 稼働時間合計
  - 自社作業員 延べ人数換算 (稼働時間 ÷ 8時間)
  - 協力業者 延べ人数合計
  - 総・延べ人数 (自社換算 + 協力業者)
  - 進捗率 (%)

### シート2: 作業項目別詳細 (全現場・全タスクの詳細)
- **対象**: 選択された現場の、すべての中間タスク（作業項目）
- **出力項目**:
  - 現場名
  - 作業項目名
  - 進捗率 (%)
  - 目標工数 (時間)
  - 消化工数 (実績・時間)
  - 自社作業員 延べ人数換算 (実績時間 ÷ 8時間)
  - 予測粗利 (項目単位の利益見込額)

## 提案する実装内容 (Proposed Changes)

### 1. AdminApp.jsx へのエクセル出力機能追加
*   **[MODIFY] `src/AdminApp.jsx`**
    *   **データ集計ロジックの追加**: 既存の `allProjectsSummary` や `summaryData` と同様のロジックを用いて、ダウンロードボタンが押された際に「現在選択されているプロジェクト」のデータを抽出して出力用のJSON配列を生成します。
    *   **xlsx ライブラリを用いたファイル生成**: 
        ```javascript
        const wb = xlsx.utils.book_new();
        const ws1 = xlsx.utils.json_to_sheet(summaryData);
        const ws2 = xlsx.utils.json_to_sheet(taskDetailData);
        xlsx.utils.book_append_sheet(wb, ws1, "現場別サマリー");
        xlsx.utils.book_append_sheet(wb, ws2, "作業項目別詳細");
        xlsx.writeFile(wb, `工数管理レポート_${today}.xlsx`);
        ```
    *   **UIの追加**: 
        - 「管理シート」（`summary`タブ）内の上部ヘッダー（項目別詳細予測の上など）に、「Excelで出力」ボタンを追加します。

## 検証計画 (Verification Plan)
### 手動検証 (Manual Verification)
1. **出力機能の実行**: AdminAppのヘッダーに追加されたエクセル出力ボタンをクリックする。
2. **ファイルのダウンロード**: `工数管理レポート_YYYY-MM-DD.xlsx` がダウンロードされることを確認する。
3. **内容の確認**: ダウンロードしたエクセルを開き、指定した項目（現場名、利益、延べ人数など）が正しく出力されていること、計算（時間÷8等の換算）が合っているか確認する。
