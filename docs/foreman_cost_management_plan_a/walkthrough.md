# 実装内容の確認 (Walkthrough) - 職長向け粗利表示（案A）

## 実装の目的
現場の職長向けに、コントロールできない間接経費（安全費、法定福利費など）をシステムから隠蔽し、純粋な直接原価（労務費ベース）とそれによる「粗利（限界利益）」のみを管理させる「案A」の運用を実現するための改修を行いました。

## 変更内容
### 1. Excelインポート時の間接費除外ロジック
Excelファイル（見積書など）を取り込む際、以下のキーワードが項目名に含まれている場合は、システムに「作業項目」として取り込まないようにフィルタリング（除外）する処理を追加しました。
- `安全費`
- `法定福利費`
- `現場管理費`
- `共通仮設費`

これにより、職長が実績入力画面や管理シートで誤って間接費の項目に時間を付けてしまうのを防ぎます。

### 2. UI文言の「粗利」ベースへの変更
管理シートにおける「損益」という表現を撤廃し、会社経費を引く前の現場利益であることを明確にするため「粗利」という言葉に変更しました。

- 「現場全体の予測**損益**」 → 「現場全体の予測**粗利**」
- 「項目別予測**損益**」 → 「項目別予測**粗利**」
- メッセージ変更:
  - 順調時: 「現在の進捗ペースなら**目標粗利**を確保可能だ！会社経費もカバーできるぞ！」
  - 赤字見込時: 「このままだと**現場粗利がマイナス**だ。至急対策を！」

## 確認手順（テスト・バリデーション）
ローカルサーバー（`npm run dev`）でアプリを開き、以下の点をご確認ください。

1. **Excelインポートのテスト**
   - 先ほどスクリーンショットでご提示いただいた「エネジェット江刺屋根改修工事」のExcelファイル（安全費や法定福利費が含まれているもの）を、再度「工事設定」画面からインポートしてみてください。
   - インポート結果に「安全費」「法定福利費」などの項目が含まれて**いない（除外されている）**ことを確認してください。
2. **管理シートの表示確認**
   - 「管理シート」タブを開き、画面上部の指標が「現場全体の予測**粗利**」になっていることを確認してください。
   - 表の右端列が「項目別予測**粗利**」になっていることを確認してください。

## 備考
現在、直接原価は「労務費（人工）」のみで計算されています。今後、材料費や外注費も直接原価として管理させたい場合は、入力欄の追加対応が可能です。
