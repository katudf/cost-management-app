# 実装計画：職長向け損益表示（案A：直接原価と粗利の管理）

## 背景と目的
安全費や法定福利費などの間接経費はシステム上（職長向け画面）には表示せず、現場の職長にはコントロール可能な「直接原価」とそれに基づく「粗利（限界利益）」のみを追わせるという「案A」の方向性で実装を行います。

## 現在のシステムの状況
現在の `App.jsx` を確認したところ、以下のように**すでに労務費（時間 × 単価）のみをベースにした予測損益計算**となっており、間接費が含まれない構造になっています。
- 予測損益 = `(目標時間 - 予測最終時間) * HOURLY_WAGE(4375円)`

したがって、**案Aを実現するための大規模なデータ構造の変更や計算ロジックの追加は不要**です。

## 提案する変更内容 (UIの明瞭化とデータ除外)
現在のシステムはすでに案Aの要件（労務費ベース）を満たしていますが、以下の改修を行うことで運用をさらにスムーズにし、職長の意識付けを強化します。

### 1. Excelインポート時の間接費除外ロジックの追加
- **対象箇所**: `App.jsx` の `handleExcelImport` メソッド
- **変更内容**: エクセルから作業項目を読み込む際、項目名に以下のキーワードが含まれる場合はマスターデータ（作業項目）として取り込まないようにフィルタリング（除外）します。
  - 除外キーワード候補: `安全費`, `法定福利費`, `現場管理費`, `共通仮設費`
- **目的**: 職長が実績入力画面や管理画面において、誤って間接費に時間をつけてしまうのを防ぐため。また、直接原価（粗利）のみを意識させるため。

### 2. UI文言の調整（「損益」から「粗利」へ）
- **「現場全体の予測損益」のラベル変更**
  - 現在: `現場全体の予測損益`
  - 変更後: `現場全体の予測粗利（直接原価ベース）` または `現場全体の予測限界利益`
- **「項目別予測損益」のラベル変更**
  - 現在: `項目別予測損益`
  - 変更後: `項目別予測粗利`
- **説明テキストの調整**
  - 現在: `現在の進捗ペースなら利益を確保可能だぜ！` / `このままだと赤字だ。すぐに対策を打とう！`
  - 変更後: `現在の進捗ペースなら目標粗利を確保可能だ！会社経費もカバーできるぞ！` / `このままだと現場粗利がマイナスだ。至急対策を！`

## 今後の拡張（オプション）
現在は「直接原価＝労務費（人工）」として計算されていますが、もし職長に「材料費」や「外注費」も直接原価として管理させたい場合は、今後のフェーズで入力欄を追加することが可能です。
