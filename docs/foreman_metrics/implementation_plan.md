## 目標 (Goal Description)
作業員アプリ（WorkerApp.jsx）の職長画面において、管理者アプリの「登録済み工事一覧」と同様に、以下の3つの指標を表示します。
1. 現場全体の予測粗利
2. 消化済工数 / 全体目標
3. 全体進捗率（または協力業者コスト）
※スマートフォンでの表示を考慮し、コンパクトで分かりやすいUIデザインとします。

## ユーザー確認事項 (User Review Required)
特にありません。

## 変更内容 (Proposed Changes)

### WorkerApp.jsx
#### [MODIFY] `src/WorkerApp.jsx`
- `useEffect` (初期ロード時):
  - `system_settings` テーブルから `hourly_wage` を取得し、ステートに保存するように追加します。
- `useEffect` (プロジェクト選択時):
  - 選択されたプロジェクトに関連する全タスクのレコード(`TaskRecords`)を（今日だけでなく全期間）取得し、合計実績時間（`totalActual`）を計算できるようにします。すでにダッシュボードで使用している `calculateProjectsSummary` ユーティリティ関数を再利用するか、同等の計算ロジックを実装します。
- UIの追加 (職長専用エリア `isForeman` ブロック内):
  - 「あなたは職長です」のメッセージの下に、以下の3つの指標を並べて表示するパネルを追加します。
    - **全体予測粗利**: `(全体目標時間 - 予測完了時間) * hourlyWage - 協力業者コスト`
    - **工数消化状況**: `全体実績時間 / 全体目標時間`
    - **全体進捗率**: （加重平均）

## 確認計画 (Verification Plan)
### 自動テスト
- `npm run build` でビルドが成功することを確認します。
### 手動検証
- ワーカーアプリを開き、職長がアサインされている現場を選択します。
- 職長専用の緑色のメッセージブロックの直下に、全体の進捗、消化工数、予測粗利が正しく表示されているか確認します。
- 管理者アプリのダッシュボードに表示されている数値と一致しているか確認します。
