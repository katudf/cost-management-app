# 現場ステータス機能とホーム画面のフィルタ・並べ替え機能 追加計画

現場（プロジェクト）にステータス（「予定」「施工中」「完了」）を付与し、さらにホーム画面（ダッシュボード）において、ステータスによる「表示／非表示の絞り込み」および「並べ替え」ができる機能を実装します。

## 背景と目的
現在、すべての現場がホーム画面に並んで表示されていますが、現場数が増えると完了した現場などに埋もれて「現在動いている現場」が把握しにくくなります。そこで、各現場にステータスを設定できるようにし、ホーム画面で柔軟に表示をコントロールできるようにします。

すでに `Projects` テーブルには `status` 列（text型）が用意されているため、これを利用します。

## User Review Required
*   **デフォルトのステータス:** 新規作成時や、過去のデータで `status` が未設定のものには、デフォルトで **「予定」** を割り当てます。
*   **並べ替えの基準:** 現在は「作成日（created_at）」の古い順などがベースになっています。今回はユーザーがUI上で「作成日順（新しい/古い）」「進捗率順」「予測粗利順」などを選べるようにする予定ですが、想定されている「並べ替え」の種類にご希望はありますでしょうか？（ひとまず一般的なものを実装します）

## Proposed Changes

### [AdminApp.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/AdminApp.jsx)

#### 1. データ取得とデフォルト値の処理
*   `fetchAllData` 関数内でのマッピング時 (`loadedProjects` 生成部分) に、DBから取得した `status` を反映します。
*   もし `status` が null や空文字の場合は、システム上で `'予定'` として扱うようにフォールバック処理を入れます。

#### 2. 「工事設定」(master) タブでのステータス変更UI
*   `activeTab === 'master'` のレンダリング領域（管理現場名や担当職長を設定する部分の下など）に、「現場ステータス」を選択できる `<select>` 要素（予定・施工中・完了）を追加します。
*   選択値が変更されたら、`updateLayer` でローカルのstateを更新すると同時に、Supabaseの `Projects` テーブルの `status` カラムを `update` します。
*   現場の新規作成 (`addNewProject`) や、Excelインポートによる新規作成時にも、初期ステータスとして `'予定'` をセットするようにします。

#### 3. 「ホーム」(dashboard) タブでのステータス表示
*   プロジェクトカード（`allProjectsSummary.map` で表示している四角い要素）の中に、現在のステータスをバッジ（色付きの小さなラベルなど）として表示します。
    *   予定（青系など）
    *   施工中（緑系や現在のアクティブカラー）
    *   完了（グレーなど）

#### 4. 「ホーム」タブでのフィルタリングと並べ替え機能の実装
*   **Stateの追加:** ホーム画面専用のstateとして以下を追加します。
    *   `filterStatus`: `['予定', '施工中', '完了']` (配列で管理。初期値はすべて含まれる、または「完了」のみ外す等の選択も可能ですが、まずは全表示をデフォルトとします)
    *   `sortOption`: `'created_asc'`, `'created_desc'`, `'progress_desc'`, `'profit_desc'` など
*   **UIの追加:** 「登録済み工事一覧」のタイトルの右側（または下）に、絞り込み用のチェックボックス（またはトグルボタン）と、並べ替え用のセレクトボックスを追加します。
*   **データ処理ロジックの変更:** ホーム画面に表示する `allProjectsSummary` をそのまま map するのではなく、追加した `filterStatus` と `sortOption` に基づいてフィルタリングとソートを行ったあとの配列（例: `filteredAndSortedProjects`）を作成し、そちらをレンダリングするように変更します。

## Verification Plan

### Manual Verification
1.  **ステータスの付与・変更テスト**
    *   「工事設定」タブを開き、任意の現場を選択する。
    *   新しく設けた「現場ステータス」のプルダウンから、「予定」「施工中」「完了」を切り替える。
    *   ブラウザをリロードし、変更したステータスが維持されていること（DBに保存されていること）を確認する。
2.  **新規作成時の初期ステータステスト**
    *   ホーム画面や工事設定画面から「新しい現場を作成」を実行する。
    *   作成された現場のステータスが、初期値として「予定」になっているか確認する。
3.  **ホーム画面での表示テスト**
    *   ホーム画面の各現場パネルに、現在のステータスがわかりやすく（バッジなどで）表示されていることを確認する。
4.  **フィルタリング（表示/非表示）テスト**
    *   ホーム画面で、「予定」「施工中」「完了」のそれぞれをオン/オフし、該当するステータスの現場だけが表示・非表示されることを確認する。
5.  **並べ替えテスト**
    *   ホーム画面で並べ替えの条件（例：新しい順、進捗率の高い順など）を変更し、現場の並び順が正しく変わることを確認する。
