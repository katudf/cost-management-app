# ホームタブのステータス別4列表示およびページネーション実装

ホームタブに表示されている工事一覧について、ステータスごとの4列表示（カンバン形式）への変更、およびページ送り機能の実装を行います。

## 背景と目的
現在、ホームタブの工事一覧はすべてのステータスが混在して横並びのグリッド要素で表示されています。
これを「見積」「予定」「施工中」「完了」の4列に分割し、それぞれ最大10件ずつ新しい順で表示することで、各ステータスの工事状況を俯瞰的に把握しやすくします。

## 変更内容

### 1. `AdminApp.jsx` のホームタブUIの刷新
*   **レイアウト変更**: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` となっていた一覧レイアウトを、`grid-cols-1 lg:grid-cols-4`（4列表示ベース）に変更します。
*   **不必要なコントロールの削除**: ステータス別にすべて表示されるため、「表示対象」の絞り込みチェックボックスを削除します。また、順番を「新しい順」に固定するため、「並び順」のプルダウンも削除します。（「工事名で検索」機能は維持します）。
*   **ページネーションステートの導入**: 各列ごとにページ番号を管理する `useState` (`{ '見積': 1, '予定': 1, '施工中': 1, '完了': 1 }`) を追加します。
*   **各列のレンダリング**: 
    1. ステータスごとのカラムヘッダー（見積、予定、施工中、完了）と件数を表示。
    2. 最大10件表示のリストとしてプロジェクトカードを一覧表示（縦並びスタック）。
    3. 全件で10件を超えるステータス列に対し、「前へ / 次へ」のページ送りボタンを配置。

### 2. `useDashboardStats.js` のデータ提供部分の改善
*   UIからコントロールが外れるため、バックエンドロジックとしての `filterStatuses` はすべて表示（全ステータス）をデフォルトとし、`sortOption` は「新しい順（`created_desc`）」で固定します。

## 検証手順
1. ブラウザでホーム画面を開く。
2. 4列表示に分かれていること、また各ステータスに該当する工事が割り振られていることを確認する。
3. いずれかのステータスが11件以上ある場合、11件目以降が次ページに切り替わり、ページ送りボタンで正しく動くことを確認する。
