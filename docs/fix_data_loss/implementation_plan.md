# データ消失防止 実装計画

シミュレーションレポートの「1. 意図しないデータ消失のリスク」を解消し、入力中のデータが失われないよう保護する機能を実装します。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> 修正後、時間が入力された状態で「作業日」や「現場」を変更しようとすると、「未送信のデータがあります」という確認ダイアログが表示されるようになります。また、入力中に通信エラーがなくても自動的にローカル端末にデータが保存（オートセーブ）され、万が一ブラウザが終了しても復元できるようになります。

## 提案される変更

### WorkerApp Component

#### [MODIFY] [WorkerApp.jsx](file:///c:/Users/katuy/Desktop/cost-management-app/src/WorkerApp.jsx)

- **状態管理の追加**
  - `hasUnsavedChanges` という状態（State）を追加し、ユーザーが時間やメモ、進捗率などを変更するたびに `true` にします。
  
- **未保存警告（画面遷移ブロック）の実装**
  - 「作業日（`selectedDate`）」と「現場の選択（`selectedProjectId`）」の `onChange` イベントにおいて、`hasUnsavedChanges` が `true` の場合は `window.confirm` で確認ダイアログを出し、キャンセルの場合は操作をブロックします。
  - 「ログアウト」時にも同様に警告を追加します。

- **オートセーブ（自動下書き保存）の実装**
  - `useEffect` を用い、入力内容（`tasks`, `subcontractors` 等）の変更を検知して1秒ごとに自動的に `localStorage` の `cost-app-unsent-draft` に保存する処理を追加します。
  - 元々の「オフライン下書き」のメッセージ表示を拡張し、オートセーブされたデータ復元の際にも専用の文言「自動保存されています」が表示されるように改修します。
  - 正常に「今日の実績を送信」ボタンで送信が完了した際に、`hasUnsavedChanges` をクリアし、下書きデータも削除します。

## 検証プラン

### 手動確認
1. WorkerApp画面でいずれかの現場を選択し、時間を入力する。
2. その状態で「作業日」または「現場」のドロップダウンを変更しようとする。
   - ⇒ 「未送信の入力データがあります。入力内容は破棄されますがよろしいですか？」のダイアログが表示されるか確認する。
   - ⇒ 「キャンセル」を押すと入力内容が維持されるか確認する。
3. 時間を入力したままブラウザ（タブ）をリロード（再読み込み）、または閉じて開き直す。
   - ⇒ 画面上部に「自動保存されたデータがあります」のお知らせが表示され、「復元」を押すことで入力途中のデータが戻ってくるか確認する。
