# データ消失リスク防止 修正内容の確認

シミュレーションレポートで指摘された「1. 意図しないデータ消失のリスク」について修正を行いました。

## 実施内容

### 未保存警告（画面遷移ブロック）の導入
ユーザーが作業項目や時間を入力した後、「今日の実績を送信」ボタンを押す前に以下の操作を行った場合、確認ダイアログ（`未送信の入力データがあります。破棄して...しますか？`）を表示してブロックするようにしました。
- 「作業日」の変更
- 「現場を選択」ドロップダウンの変更
- 上部の「入力済みの現場」ショートカットからの現場切り替え
- ログアウト操作

### オートセーブ（自動下書き保存）機能の追加
- ユーザーが入力操作（時間、メモ、進捗率、協力業者などの変更）を行うたびに、約1秒の遅延（デバウンス処理）でブラウザのローカルストレージに自動的にデータを保存する仕組みを組み込みました。
- これにより、誤って**ブラウザのタブを閉じてしまった**、あるいは**アプリが強制終了**してしまった場合でも、再度アプリを開いた際に「自動保存されたデータがあります」と通知され、入力途中の状態を「復元」できるようになります。

## 確認事項
- [x] 未送信の状態（時間を入力したまま）で日付や現場を変更しようとすると、警告ダイアログが出るようになったこと。
- [x] 未送信の状態でログアウトボタンを押すと、同様に警告が出るようになったこと。
- [x] これらの警告で「キャンセル」を選ぶと、入力状態がそのまま維持されること。
- [x] 作業中のデータが自動保存され、ページをリロードした際に「自動保存されています」というお知らせが表示されること。
- [x] 復元ボタンを押すと、入力途中のフォームが完全に復元されること。
