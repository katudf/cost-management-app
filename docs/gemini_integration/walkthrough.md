# Gemini API トークン利用量監視機能の追加

AI最適化に関連して、Googleが定めるGemini APIの無料枠制限（リクエスト回数および使用トークン数）を意識しながら運用できるよう、利用状況のローカル集計・可視化機能を追加実装いたしました。

## 実装した機能の概要

1. **トークン情報の記録**
   AIから応答が返ってきた際に含まれる `usageMetadata` （そのリクエストで消費した合計トークン数）を取得し、ブラウザのローカルな記憶領域に自動記録するようにしました。
2. **日付ごとのリセットロジック**
   その日の午前0時を越えると自動でカウントが「0」に戻るため、1日ごとの利用目安「例えば1500回制限」などを正しく把握できます。
3. **モニタリング用専用パネルの追加**
   『システム設定』タブの「Gemini API キー」を入力する欄の下に、現在の使用状況をメーター（プログレスバー）とともに表示するパネルを追加しました。

## 動作確認手順

1. 『システム設定』タブを開きます。
2. 「Gemini API キー」の入力欄の下に、青い枠で囲まれた **本日のAPI利用状況 (ローカル集計)** という項目が表示されていることを確認します。
3. 先ほどお試しいただいた際に消費した「リクエスト回数（○回）」と「消費トークン数（○トークン）」がリアルタイムで反映されているはずです。
4. インポート処理を再度テストで行い、完了後にこの画面へ戻ってくるとメーターが増加していることが確認できます。
※ メーターが全体の80%（上限の8割）を超えた場合は、警告として赤色に変化するよう設定しています。

## 追加の改善点: 任意でのAI実行への切り替え（エラー回避機能）

Google側のサーバー混雑等によりAI側から `[503] サービス利用不可` のエラーが返ってきた場合でも、システムとしての通常インポート業務が止まってしまわないよう、仕様を見直しました。
- Excel読み込み直後に自動でAIを走らせるのではなく、確認画面にて**「項目名をAIで自動整理しますか？」というパネル**が出るように変更しました。
- 急ぎの場合やAIの使用制限に引っかかった場合はそのまま「現在の現場を上書き」等の通常通りのインポートが可能です。
- AIによる整理が必要な場合のみ「AIを使用して項目を整理する」ボタンを押すことで、任意のタイミングで最適化をご利用いただけます。

## 重要な変更ファイル

- `src/utils/aiOptimizeUtils.js` - APIの呼び出し関数の拡張と、消費量ローカル集計・取得関数の追加
- `src/components/tabs/SystemSettingsTab.jsx` - UI実装（利用状況表示の専用パネルおよびメーター追加）
- `src/AdminApp.jsx` & `src/components/ImportModal.jsx` - AI自動実行からユーザー選択式（任意実行）へのロジック・UI変更
