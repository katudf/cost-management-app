# Gemini API 利用量モニタリング機能の追加

無料枠（1分あたり15リクエスト、1日あたり1500リクエスト、100万トークン等 ※Googleの規定による）の超過を防ぎ、現在の使用状況を把握するための「監視・可視化機能」を追加します。

## 実装のアプローチ

Geminiからの応答データには `usageMetadata`（消費したトークン数に関するデータ）が含まれています。これを利用して、システム内でリクエスト毎の消費を記録し、画面に表示させます。

## User Review Required

実装を進める上で、**利用データの保存先**について以下いずれが良いかご指定をお願いします。

> [!IMPORTANT]
> **A方式: ブラウザ（ローカル）に保存して集計する**
> データベースの変更が不要で簡単です。ただし「作業中のご自身のPC」単位での集計になります。別のPCから他の方がインポートした場合は、そのPCごとに別々にカウントされます。小規模運用や一時的な目安ならこれで十分なことが多いです。
> 
> **B方式: データベース（Supabase）に保存して集計する**
> アカウント全体（全利用者の合算）を正確にモニタリングできます。ただし、先ほどと同様にSupabase側で新しくテーブルを追加作成するSQLコマンドを実行していただく必要があります。

現在のシステムの特性としては**A方式（ローカル集計）**でも、1日の限度額を越えるような膨大な連続インポートがなければ十分に目安として機能するかと思いますが、いかがでしょうか？（ご希望の方式をお知らせください！）

## Proposed Changes

### 1. トークン数の取得と記録処理の実装
#### [MODIFY] src/utils/aiOptimizeUtils.js
- モデルからのレスポンス結果 (`result.response.usageMetadata`) を解析し、「消費した入力トークン数」「出力トークン数」「合計トークン数」を取得できるようにします。
- 取得した数値を、ローカルストレージ（またはDB）に「今日の日付」と一緒に＋1ずつ合算・記録するロジック（または関数呼び出し）を追加します。

### 2. 利用状況の可視化（UI）
#### [MODIFY] src/components/tabs/SystemSettingsTab.jsx
- システム設定タブの「APIキー入力欄」の横、あるいは下部に、新しく **「本日のAPI利用状況」** というカード（パネル）を追加します。
- **表示内容:**
  - 本日のリクエスト回数（例: 3回 / 目安1500回）
  - 本日の消費トークン数（例: 24,500トークン / 目安100万トークン）
- メーター（プログレスバー）等を活用して、視覚的に上限到達の危険度（緑・黄・赤）が分かるようにデザインします。

## Verification Plan

2. 日付をまたいだ際（または強制的に日付バグを発生させた際）に、カウンターが正しく0リセット（本日の消費としてリフレッシュ）されるかを検証します。
