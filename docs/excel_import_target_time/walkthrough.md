# 実装内容の確認 (Walkthrough) - Excelインポート時の目標時間自動設定

## 実装の目的
Excelの見積書をインポートした際、各作業項目の「目標時間（target）」が一律で0になってしまう問題を解消し、見積書の「金額」データを基に、システム側で自動的に目安となる「目標時間（人工）」を逆算して設定する機能を実装しました。

## 変更内容
### App.jsx のインポート処理 (`handleExcelImport`) の改修
1. **金額列の取得**
   - ご連携いただいた情報に基づき、Excelの **「AX列（インデックス49）」** を「金額」列として取得するように設定しました。
2. **目標時間の自動計算ロジック**
   - 設定されている職長単価（`HOURLY_WAGE = 4375`円 / 1時間）を使用し、`金額 ÷ 4375` でその作業にかけられる「目標時間」を算出します。
   - 計算結果の小数点以下は **四捨五入（`Math.round`）** し、整数として設定します。
   - ※金額が数値として読み取れない場合、または金額が0以下の場合は、従来通り目標時間は `0` となります。

## 確認手順（テスト・バリデーション）
ローカルサーバー（`npm run dev`）でアプリを開き、以下の点をご確認ください。

1. **Excelファイルのインポート**
   - 「工事設定」タブから、普段お使いのExcel見積書をインポートします。
2. **目標時間の確認**
   - インポート完了後、「工事設定」画面の「仕様項目ごとの目標設定」エリアにおいて、各項目の「目標時間」に自動で数字（時間）が入力されていることを確認してください。
   - 電卓などで、**「Excel上のその項目の金額 ÷ 4375」の四捨五入** の値と一致しているか、いくつかの項目でテストをお願いいたします。
   - （例：Excel上の金額が 50,000円 の場合、50000 ÷ 4375 = 11.42... なので、目標時間が `11` h となっていれば正常です）

もし金額列の読み取りがうまくいかない（すべての目標時間が0のままになる）などの挙動があれば、Excelのフォーマット（AX列が本当に数値データとして入っているかなど）を再調査いたしますのでお申し付けください。
