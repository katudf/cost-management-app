# 実装計画：Excelインポート時の目標時間（target）自動設定

## 背景と目的
現在、Excel（見積書）をインポートした際、各作業項目の「目標時間（`target`）」は一律で `0` として取り込まれています。これを改善し、インポート時に見積書の「金額」データを活用して、自動的に目標時間を設定（試算）できる機能を実装します。

## ロジック設計
現在のシステムでは、職長の労務費単価（時間ベース）として `HOURLY_WAGE = 4375`（1日8時間換算で35,000円）が定義されています。
見積書に記載されている各項目の「金額（金額もしくは単価×数量）」を、この時間単価で割り戻すことで、その作業項目に割り当て可能な「目標時間」を算出します。

### 仕様の検討
1. **Excel列からの「金額」の取得:**
   - App.jsx の `handleExcelImport` を確認すると、現在以下のセルを取得しています。
     - `name = row[2]`
     - `spec = row[24]`
     - `quantity = row[40]`
     - `unit = row[43]`
   - 見積書のフォーマット上、単価または金額が入っている列（たとえば `row[46]` や `row[50]` など）の特定が必要です。一般的な配置を想定し、「金額」列を取得するよう改修します。

2. **目標時間の計算式:**
   - 取得した金額（例: 50,000円） / `HOURLY_WAGE` (4375円)
   - 小数点以下は四捨五入（または切り捨て/切り上げ）し、整数（時間）として `target` にセットします。
     - 例: 50,000 / 4375 = 11.42... → 目標: 11時間

### App.jsx (`handleExcelImport`) の修正内容
- Excelのパースループ内で金額列（例として `amount = row[金額列インデックス]`）を取得。
- 金額が有効な数字であれば、`Math.round(amount / 4375)` で目標時間を計算。
- 金額データがない、または0の場合は、従来通り `target: 0` とする。
