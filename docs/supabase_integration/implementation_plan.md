# 計画：Supabaseの従業員データの活用

## 概要
別プロジェクトで使用している Supabase の従業員データを参照し、本アプリの「日報実績入力」画面での「名前」入力時に、そのデータを活用（選択・サジェスト）できるようにします。

## 実装ステップ

### 1. Supabase クライアントの導入
`@supabase/supabase-js` パッケージをインストールし、環境変数に基づいたクライアント初期化ファイル (`src/lib/supabase.js`) を作成します。

### 2. 環境変数の設定
プロジェクトルートに `.env.example` （及び実際の `.env`）を作成し、以下のような変数を定義します。
```env
VITE_SUPABASE_URL=あなたのSupabaseプロジェクトURL
VITE_SUPABASE_ANON_KEY=あなたのSupabaseのAnonキー
```

### 3. App.jsx の修正
#### [MODIFY] App.jsx
- 初期マウント時（またはデータフェッチ用フック内）で、Supabaseの `Workers` テーブルから名前リストを取得し、ステート（ `const [employees, setEmployees] = useState([]);`）に保持します。並び順は `display_order` または `order` 順を考慮します。
- 「日報実績入力」タブ内のレコード入力部分にある `worker`（名前）の入力フィールドを、 `<datalist>` と組み合わせた入力欄に変更します。
  これにより、取得した従業員名を候補としてサジェストしつつ、新しい名前を手入力することも可能な柔軟なUIとなります。

## 検証計画
### 手動検証
1. プロジェクトルートに `.env` を作成し、正しい情報をセットする。
2. アプリを起動し、日報実績入力画面を開く。
3. 名前のテキストボックスをクリックした際に、Supabaseから取得した従業員名がサジェストとして表示されることを確認する。
