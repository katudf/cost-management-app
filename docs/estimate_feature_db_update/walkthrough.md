# 見積書機能に伴うデータベース定義書の更新 - 修正内容の確認

見積書作成機能の実装に伴うデータベースの変更（SQLマイグレーション）を反映し、`docs/database/database_info.md` を最新の状態に更新しました。

## 実施した変更内容

### 1. 新規テーブルの定義追加
- **`estimates` (見積ヘッダー)**: 見積番号、顧客情報、発効日、ステータス、合計表示設定などの基本情報の定義を追加。
- **`estimate_items` (見積明細)**: 項目種別（カテゴリ/明細/小計）、数量、単価、および階層構造を管理する定義を追加。

### 2. 既存テーブルへのカラム追加
- **`Workers`**: 印影画像用の `stamp_url` を追加。
- **`system_settings`**: 会社情報（社名、住所、TEL/FAX等）、社印・代表印・承認印のURL、および見積表のデフォルト表示設定に関するカラムを追加。

### 3. ビューおよび自動演算ロジックの追記
- **`v_estimate_category_totals`**: 見積書内のカテゴリ名と、そのカテゴリに属する明細の合計金額を算出するビューの定義を追加。
- **トリガーと関数**: 金額の自動計算トリガー、更新日時の自動更新、および見積番号の自動採番関数についての説明を「5. ビュー・関数・トリガー」セクションとして追加しました。

## 検証結果

- [x] 提供されたSQLのすべてのテーブル、カラム、データ型、制約がドキュメントに正しく反映されていることを確認。
- [x] `estimates` ↔ `Customers`, `Projects`, `Workers` などの外部キーリレーションの記載を確認。
- [x] ドキュメントのセクション番号を振り直し、階層構造を整理。
