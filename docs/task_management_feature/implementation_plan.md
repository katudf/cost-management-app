# 実装計画: 作業項目（タスク）の直接追加・削除機能

## 概要
現在、工事設定（マスター管理）画面内の作業項目（`ProjectTasks`）は、最初に追加された仮データか、Excelからの一括インポートによってのみリスト化されています。
本機能では、Excelがなくても画面上から直接個別の作業項目を手動で「追加」「削除」できるボタンおよび機能を実装します。

## 変更内容

### 1. 新規追加機能 (`addMasterItem`)
- `App.jsx` 内に新しい関数 `addMasterItem` を作成します。
- この関数は、以下の処理を行います：
  1. Supabaseの `ProjectTasks` テーブルへ、デフォルト値（名前＝"新規作業項目", 目標時間＝0）をもつ新規レコードを `INSERT` します。このとき、並び順（`order`）として `activeProject.masterData.length + 1` を渡します。
  2. データベースへの登録が成功したら取得した新しいIDを使い、ローカルステート（`projects` -> `masterData`）へ行を追加します。
- UIの「工事設定」タブ内のテーブル下部（合計行の下）に、「＋ 新しい作業項目を追加」ボタンを設置します。

### 2. 削除機能 (`removeMasterItem`)
- `App.jsx` 内に新しい関数 `removeMasterItem` を作成します。
- 引数として対象の `taskId` を受け取ります。
- この関数は、以下の処理を行います：
  1. JavaScriptの `confirm()` ダイアログで「この作業項目を削除すると、紐づく実績データもすべて削除されます。本当によろしいですか？」と確認を取ります。
  2. OKが押されたら、Supabaseの `ProjectTasks` テーブルから該当の行を `DELETE` します。
      > ※注意点：Supabase側で `FOREIGN KEY ... ON DELETE CASCADE` が設定されていれば `TaskRecords` も自動で消えますが、念のためUI（ローカルステート）から該当実績行も消去します。
  3. ローカルステートから対象の `masterData` および関連する `records` をフィルタリングして削除します。
- UIの各作業項目行の右端（現在設定ボタンなどがある部分の横）に、「ゴミ箱」アイコンの削除ボタンを設置します。

## 検証計画
- `npm run dev` でローカルサーバーを起動。
- 「工事設定」タブを開き、「＋ 新しい作業項目を追加」を押すと新しい行が追加されること、その行の名前などを変更して自動保存されることを確認。
- 既存の作業項目の行のゴミ箱ボタンを押し、警告ダイアログでOKを押すと、その行が消えること（およびリロード後も消えていること）を確認。
