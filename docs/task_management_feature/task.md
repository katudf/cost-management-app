# 工事設定画面のタスク追加・削除機能

- [ ] 1. 事前準備と設計
  - [ ] 1.1 `App.jsx` 内のマスターデータ（`ProjectTasks`）操作ロジックを確認。
- [ ] 2. 機能の実装（React State & Supabase DB）
  - [ ] 2.1 **新規追加機能**: `addMasterItem` 関数を作成する。
    - Supabaseの `ProjectTasks` に新しい行を INSERT。
    - 成功後、ローカルステート（`projects`）の `masterData` に新しいタスクを追加。
  - [ ] 2.2 **削除機能**: `removeMasterItem` 関数を作成する。
    - ユーザーに確認ダイアログを表示（「このタスクと紐づく実績も削除されますがよろしいですか？」）。
    - Supabaseの `ProjectTasks` から該当IDを DELETE。
    - 成功後、ローカルステートの `masterData` と `records` から関連データを削除。
- [ ] 3. UIの改修（工事設定タブ）
  - [ ] 3.1 テーブルの下（または上）に「＋ 新しい作業項目を追加」ボタンを配置し、`addMasterItem` を紐付ける。
  - [ ] 3.2 各タスク行の右端（現在ゴミ箱アイコンがあるか確認）に削除ボタンを配置し、`removeMasterItem` を紐付ける。
- [ ] 4. テストと動作確認
  - [ ] 4.1 新規追加ボタンで空のタスク行が追加され、名前や目標時間を編集・保存できるか確認。
  - [ ] 4.2 削除ボタンでタスクが消え、データベース上からも削除されるか確認。
