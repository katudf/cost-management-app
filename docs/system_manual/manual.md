## 1. システム概要

本システムは、建設現場の**工事原価管理**と**作業員の日報・配置管理**をWebブラウザ上で一元的に行うためのツールです。

### 主な機能

| 機能カテゴリ | 内容 |
|---|---|
| **工事原価管理** | 見積金額・目標工数の設定、進捗管理、予測粗利の自動算出 |
| **作業日報** | 作業員がスマホから日々の作業時間・進捗を入力 |
| **配置表** | 8週間分の作業員配置をガントチャート形式で管理 |
| **帳票出力** | 就労日報（PDF/Excel）、管理シート（Excel）、配置表（Excel） |
| **仕入帳** | 材料の購入記録管理 |
| **作業員管理** | 名簿・資格情報・表示順の管理 |
| **AI連携** | Excelインポート時に見積項目名をAIで自動最適化 |

### システム構成

- **フロントエンド**: React + Vite (TailwindCSS)
- **バックエンド / データベース**: Supabase (PostgreSQL)
- **外部API**: Google Gemini API（任意）

### 3つのアプリケーションモード

本システムは1つのURLに対し、クエリパラメータで3つのモードを切り替えて利用します。

| モード | URL | 用途 | 対象ユーザー |
|---|---|---|---|
| **管理者画面** | `/?mode=admin` またはパラメータなし | すべての管理機能 | 管理者 |
| **作業員画面** | `/?mode=worker` | 日報入力（スマホ向け） | 作業員・職長 |
| **配置予定表** | `/?mode=schedule` | 配置表の閲覧（読み取り専用） | 作業員・職長 |

## 2. 動作環境・アクセス方法

### 推奨動作環境

| 項目 | 推奨環境 |
|---|---|
| ブラウザ | Google Chrome（最新版）、Microsoft Edge |
| PC画面解像度 | HD（1366×768）以上 |
| モバイル | iOS Safari / Android Chrome |

### 初期セットアップ

1. `.env` ファイルに Supabase の接続情報を設定します。

```
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJh...（APIキー）
```

2. 依存パッケージをインストールします。

```bash
npm install
```

3. 開発サーバーを起動します。

```bash
npm run dev
```

4. ブラウザで `http://localhost:5173/` を開きます。

## 3. 管理者画面（AdminApp）

管理者画面では上部のナビゲーションバーから各タブを切り替えて操作します。

### ナビゲーション構成

| タブ名 | アイコン | 機能概要 |
|---|---|---|
| **ホーム** | 🏠 | 登録済み工事一覧・進捗サマリー |
| **工事設定** | ⚙️ | 工事の基本設定・管理シート・実績入力 |
| **作業員** | 👥 | 作業員マスター管理・日報出力 |
| **配置表** | 📅 | 作業員配置のガントチャート |
| **設定** | ⚙️ | システム共通設定・休日・資格管理 |
| **材料** | 📄 | 仕入帳データの管理 |

---

### 3.1 ホーム（ダッシュボード）

登録されているすべての工事をカード形式で一覧表示します。

#### 表示項目（カード）

各工事カードには以下の情報が表示されます。

- **現場名**
- **ステータス**（見積 / 予定 / 施工中 / 完了）
- **担当職長名**
- **全体進捗率**（プログレスバー）
- **消化工数 / 目標工数（h）**
- **予測粗利**（黒字は緑、赤字は赤で表示）

#### フィルタ・ソート機能

| フィルタ・ソート | 説明 |
|---|---|
| **表示対象** | 見積・予定・施工中・完了をチェックボックスで絞り込み |
| **並び順** | 登録日（新/古）、進捗率（高/低）、予測粗利（高/低）から選択 |

#### 操作方法

- **カードをクリック** → 該当工事の「工事設定」タブに遷移します。
- 工事が未登録の場合、「新しい現場を作成」ボタンが表示されます。

---

### 3.2 工事設定タブ

工事設定タブは、さらに3つのサブタブに分かれています。

#### 3.2.1 工事基本設定

工事の基本情報と仕様項目を設定します。

##### 基本情報

| 項目 | 説明 |
|---|---|
| **管理現場名** | 現場の名称。変更すると自動保存されます。 |
| **現場ステータス** | 見積 → 予定 → 施工中 → 完了 の順でワークフロー管理 |
| **担当職長** | 登録済み作業員から選択。職長は作業員アプリで進捗入力が可能になります |
| **工期 開始日 / 終了日** | 配置表のバーチャート表示に使用されます |
| **配置表カラー** | 配置表での現場の表示色を選択。16色のカラーパレットから選択可能 |

##### 仕様項目ごとの目標設定

各行が1つの作業項目（仕様項目）に対応しています。

| 項目 | 説明 |
|---|---|
| **作業項目（仕様）** | 仕様の名称（例：「屋根：ウレタン防水」） |
| **見積金額** | この項目の見積金額（¥）。入力すると目標時間が自動計算されます |
| **目標時間** | `見積金額 ÷ 人工単価（時間単価）` で自動算出。読み取り専用 |

> [!TIP]
> **Excelからインポート**: 「Excelからインポート」ボタンで、見積書のExcelファイルから仕様項目を一括取り込みできます。

##### 新しい現場を追加

- ヘッダー左の「+ 新しい現場を追加」ボタンで空の工事を作成できます。

##### 現場を削除

- 「現場を削除」ボタンで、選択中の工事データをすべて削除（確認ダイアログあり）します。

#### 3.2.2 管理シート

工事の原価管理シートを表示します。

##### 表示要素

| 表示 | 説明 |
|---|---|
| **現場全体の予測粗利** | 全作業項目の予測粗利合計。赤字の場合は赤背景で点滅表示 |
| **消化済工数 / 全体目標** | 工事全体の実績時間と目標時間の比較 |
| **協力業者 発生コスト** | 協力業者への支払い累計額 |

##### 項目別詳細予測テーブル

| 列名 | 説明 |
|---|---|
| 作業項目 | 仕様項目名 |
| 見積金額 | 設定済みの見積金額 |
| 目標/実績 | 目標時間と消化済み時間 |
| 進捗 | スライダーで 0〜100% を 5% 刻みで調整可能 |
| 工数差異 | 目標対比の差異率（%） |
| 項目別予測粗利 | 現在のペースで完了した場合の予測損益 |

> [!IMPORTANT]
> **予測粗利の計算ロジック**: `進捗率 > 0` の場合、`予測最終工数 = 実績 ÷ (進捗率/100)` として計算し、`予測粗利 = (目標時間 - 予測最終工数) × 時間単価` で算出されます。

- **Excel出力** ボタンで管理シートをExcelファイルとしてダウンロードできます。

#### 3.2.3 実績入力

管理者が直接、日報実績を入力・修正するための画面です。

##### 日報実績入力

| 項目 | 説明 |
|---|---|
| 日付 | 作業日（日付選択） |
| 作業 | 仕様項目から選択 |
| 名前 | 作業員名（入力中にリストから候補選択可能） |
| 時間 | 作業時間（h） |
| 備考 | フリーテキストのメモ |

**絞り込み機能**: 日付・氏名で表示をフィルタできます。

##### 協力業者実績入力

| 項目 | 説明 |
|---|---|
| 日付 | 作業日 |
| 会社名 | 協力業者名 |
| 人数(人) | 作業人数（小数可、0.1単位） |
| 入力者 | 入力した職長名（任意） |

### 3.3 作業員タブ

作業員の名簿管理と日報出力を行います。左右2カラム構成です。

#### 左側：作業員マスター設定

| 操作 | 説明 |
|---|---|
| **追加する** | 新しい作業員を登録 |
| **ドラッグ＆ドロップ** | カードをドラッグして表示順を変更。この順番が作業員アプリのログイン画面に反映されます |
| **カードをクリック** | 作業員詳細モーダルが開きます（編集・削除が可能） |

##### 作業員詳細情報

| 項目 | 説明 |
|---|---|
| 氏名 | 作業員の名前 |
| フリガナ | カナ表記 |
| 生年月日 | 年齢が自動計算されます |
| 連絡先 | 電話番号やメールアドレス |
| 退職日 | 設定するとリストから非表示になります |

> [!NOTE]
> 「退社済みの作業員を表示」チェックを入れると退職者も表示されます。

#### 右側：作業員別 日報出力

作業実績のある作業員がリストで表示されます。

| 操作 | 説明 |
|---|---|
| チェックボックス | 出力対象者を選択（全選択/個別選択） |
| **「N名分を出力」** ボタン | チェックした作業員の日報出力モーダルを起動 |

##### 日報出力モーダル

| 項目 | 説明 |
|---|---|
| 出力する週 | 月曜始まりの週を日付で選択 |
| **PDF出力** | ブラウザ印刷ダイアログ経由でA4横（就労日報）のPDFを出力 |
| **Excel出力** | `.xlsx` ファイルとしてダウンロード |

> [!TIP]
> 複数名分を一度にチェックして一括出力が可能です。

### 3.4 配置表タブ

作業員の日別配置をガントチャート形式で管理します。**8週間分**（56日間）を一覧表示します。

#### 画面構成

```
┌─────────────────────────────────────────────────┐
│ ナビゲーション: ← 今日 表示期間 →                    │
├─────────────────────────────────────────────────┤
│ 会社行事/休日 行                                    │
├─────────────────────────────────────────────────┤
│ 現場バーチャート（工期表示）                          │
├─────────────────────────────────────────────────┤
│ 作業員 × 日付 の配置セル（メインエリア）               │
└─────────────────────────────────────────────────┘
```

#### 現場バーチャート

- 工事設定で「工期 開始日/終了日」が設定された現場が色付きバーで表示されます。
- **ドラッグ＆ドロップ**で行の並び順を変更できます（並び順はDBに保存されます）。
- バーにマウスホバーすると**現場の概要ツールチップ**（進捗率・消化工数など）が表示されます。
- 現場名クリックで工事設定タブに遷移します。

#### 配置セルの操作

| 操作 | 説明 |
|---|---|
| **セルをクリック** | 配置編集ポップアップが開きます |
| **ドラッグ選択** | 横方向にドラッグして複数日を同時選択 → まとめて配置追加 |
| **Ctrl+C** | 選択セルの配置をコピー |
| **Ctrl+V** | コピーした配置をペースト（1日→複数日の展開にも対応）|
| **Ctrl+X** | 選択セルの配置をカット |
| **Delete / Backspace** | 選択セルの配置を一括削除 |

##### 配置編集ポップアップ

- **工事一覧**: 「予定」「施工中」ステータスの工事が表示され、クリックで配置追加
- **スケジュール種別**: 有給、休み、健診、会社、講習会、会議、CPDS から選択
- **既存配置**: ドラッグで並び替え、✕ボタンで個別削除が可能

#### 会社行事/休日 行

- 最上段の行のセルをクリックすると、会社行事や休日を設定するダイアログが開きます。
- 休日設定した日は、列全体が**赤背景**で表示されます。

#### Excel出力

- 配置表をExcelファイルとしてダウンロードできます（画面右上の「Excel出力」ボタン）。

#### 閲覧専用リンク

- 「閲覧用を開く」リンクから `?mode=schedule` の配置予定表（読み取り専用）を別タブで開けます。

### 3.5 設定タブ

システム全体の共通設定を管理します。

#### 3.5.1 基本設定

| 設定項目 | 説明 |
|---|---|
| **1人あたりの人工費用** | 円/時間。目標時間の自動計算や予測粗利に使用されます。既定値: ¥3,500/h |
| **Gemini API キー** | Google AI Studioで取得したAPIキー。Excelインポート時のAI最適化機能に使用 |

> [!IMPORTANT]
> 人工費用を変更した場合、既存の工事の「目標時間」は自動で再計算されません。変更後は各工事の見積金額を再入力するか、手動で調整してください。

#### 3.5.2 会社休日カレンダー

- 月単位のカレンダーから日付をクリックして会社休日を設定・解除します。
- 設定した休日は配置表や就労日報PDF（赤背景表示）に反映されます。

#### 3.5.3 資格情報マスター

作業員に紐づく資格情報を管理します。

| 項目 | 説明 |
|---|---|
| 作業員名 | 資格保持者を選択（必須） |
| 資格名 | 資格の名称（必須。例: 職長・安全衛生責任者） |
| 登録番号 | 資格の登録番号（任意） |
| 取得日 | 資格を取得した日付（任意） |
| 有効期限 | 資格の有効期限（任意） |

- 一人の作業員に複数の資格を登録できます。
- テーブルでは作業員ごとにグループ化表示され、クリックで展開/折りたたみ可能です。

#### 3.5.4 API利用状況

Gemini APIの本日の利用状況（リクエスト回数・消費トークン数）がゲージで表示されます。

## 4. 作業員画面（WorkerApp）

スマートフォン向けに最適化された日報入力画面です。`?mode=worker` でアクセスします。

### 4.1 ログイン

- 登録済みの作業員名がボタン一覧で表示されます。
- 自分の名前をタップしてログインします（パスワード不要）。
- ログイン状態はブラウザのローカルストレージに保存されます。

> [!NOTE]
> 退職日が設定されている作業員はログイン画面に表示されません。

### 4.2 日報入力の流れ

#### ① 作業日を選択

- デフォルトは当日の日付。過去日の入力も可能です。
- 日付の下に**季節区分**（夏季/冬季）と**定時時間**が表示されます。

#### ② 入力済み現場の表示

- その日にすでに入力済みの現場がバッジで表示されます。
- バッジをタップするとその現場に切り替わります。

#### ③ 現場を選択

- ドロップダウンから作業する現場を選択します。

#### ④ 作業項目ごとに時間を入力

各作業項目がカードで表示されます。

| 入力項目 | 説明 |
|---|---|
| **開始時刻** | 15分刻みのtime入力 |
| **終了時刻** | 15分刻みのtime入力 |
| **メモ** | フリーテキスト（任意） |

##### 自動計算バッジ

時刻を入力すると以下が自動計算されて表示されます。

- **実働時間**（休憩控除後）
- **時間外労働時間**（早出・残業）
- **控除された休憩時間**

##### 複数時間帯の入力

- 「時間帯を追加」ボタンで、1つの作業項目に複数の時間帯を登録できます。
- 例：午前と午後に同じ作業を行った場合など。
- 複数スロットがある場合は合計値も表示されます。

##### 新規作業項目の追加

- 「新しい作業項目を追加」ボタンで、その場で新しい仕様項目を管理DBに追加できます。

#### ⑤ 時間帯重複チェック

- 異なる現場間で作業時間が重複している場合、**赤色の警告バナー**が表示されます。
  - 例：「A現場」と「B現場」の作業時間が 8:00〜10:00 で重複

#### ⑥ 職長ダッシュボード（職長のみ）

現場の担当職長としてログインしている場合、画面上部に**職長ダッシュボード**が表示されます。

| 表示 | 説明 |
|---|---|
| 全体進捗率 | 現場全体の加重平均進捗率 |
| 消化工数 / 全体目標 | 実績時間 / 目標時間 |
| 現場全体の予測粗利 | 現在のペースでの予測損益（協力業者コスト控除後） |

- 職長は各作業項目の**進捗率スライダー**（0〜100%、5%刻み）を操作可能です。

#### ⑦ 協力業者入力（職長のみ）

職長のみが表示される「協力業者(常用)」セクションで、当日の協力業者を入力できます。

| 入力項目 | 説明 |
|---|---|
| 会社名 | 協力業者名 |
| 人数 | 作業人数（0.1人単位） |

#### ⑧ 送信

画面下部の「今日の実績を送信」ボタンをタップして保存します。

- 送信ボタンには現場別の合計時間・人工数が表示されます。
- 複数現場に入力がある場合、本日の全体合計も表示されます。

## 7. AI機能（Gemini連携）

### 概要

Excelファイルから仕様項目をインポートする際に、Google Gemini APIを使用して項目名の自動最適化を行う機能です。

### 設定方法

1. [Google AI Studio](https://aistudio.google.com/) でAPIキーを取得
2. 「設定」タブの「Gemini API キー」に入力して保存

### 機能詳細

| 機能 | 説明 |
|---|---|
| **項目名の最適化** | 長い仕様名を短く分かりやすく自動短縮 |
| **不要項目の除外** | 直接工事費、工事価格など集計行を自動で除外候補にマーク |

### 利用制限

- 1日のリクエスト数・トークン数に上限があります（ローカルで集計）。
- APIが混雑している場合（503エラー等）、時間を置いてから再試行してください。

### インポートの流れ

1. 「工事設定」→「Excelからインポート」で見積書Excelを選択
2. 項目一覧がプレビュー表示されます
3. 「AIで最適化」ボタンをクリック（任意）
4. 最適化結果を確認し、各項目の除外/含むを調整
5. インポート方法を選択:
   - **新規現場として追加**: 新しい工事を作成
   - **既存現場に上書き**: 選択中の工事のデータを置き換え
   - **別名で追加**: 同名の工事が存在する場合に別名で作成

## 9. 勤務時間計算ルール

本システムでは、開始・終了時刻から実労働時間と時間外労働を自動計算します。

### 季節区分

| 区分 | 対象月 | 定時 | 定時実働 |
|---|---|---|---|
| **夏季** | 3月〜10月 | 8:00〜17:30 | 7.5時間 |
| **冬季** | 11月〜2月 | 8:00〜17:00 | 7.5時間 |

### 休憩時間ルール

#### 夏季（3月〜10月）

| 休憩 | 時間帯 | 分数 |
|---|---|---|
| 午前休憩 | 10:00〜10:30 | 30分 |
| 昼休憩 | 12:00〜13:00 | 60分 |
| 午後休憩 | 15:00〜15:30 | 30分 |
| **合計** | | **120分** |

#### 冬季（11月〜2月）

| 休憩 | 時間帯 | 分数 |
|---|---|---|
| 午前休憩 | 10:00〜10:15 | 15分 |
| 昼休憩 | 12:00〜13:00 | 60分 |
| 午後休憩 | 15:00〜15:15 | 15分 |
| **合計** | | **90分** |

### 計算方法

1. **総分数** = 終了時刻 − 開始時刻
2. **休憩控除** = 勤務時間帯と重複する休憩時間の合計
3. **実労働時間** = 総分数 − 休憩控除
4. **時間外労働** = 早出時間（定時開始前の実作業）+ 残業時間（定時終了後の実作業）
5. **人工数** = 実労働時間 ÷ 7.5時間

### 計算例（夏季、8:00〜18:00の場合）

```
総分数: 600分 (10h)
休憩控除: 30 + 60 + 30 = 120分
実労働時間: 480分 (8.0h)
時間外: 17:30〜18:00 = 30分 (0.5h)
人工: 8.0 ÷ 7.5 = 1.1
```

## 10. トラブルシューティング

### よくある問題と対処法

| 症状 | 原因 | 対処法 |
|---|---|---|
| 画面が表示されない | Supabase接続エラー | `.env`ファイルの接続情報を確認 |
| PDF出力でポップアップがブロックされる | ブラウザのポップアップブロック設定 | ブラウザの設定でポップアップを許可 |
| Excelインポートでデータが見つからない | Excelファイルのフォーマット不一致 | 所定のフォーマットに準拠したExcelを使用 |
| AIの最適化に失敗する（503エラー） | Google APIサーバーの混雑 | 時間を置いて再試行 |
| AIの最適化に失敗する（404エラー） | AIモデルが廃止された可能性 | 別のモデルを使用するか、AI最適化をスキップ |
| 配置表の並び替えが保存されない | UNIQUE制約による競合 | ページをリロードして再度並び替え |
| 作業員アプリで作業員が表示されない | 退職日が設定されている | 管理者画面で退職日を確認 |
| 作業時間の重複警告が出る | 異なる現場で同時間帯が入力されている | いずれかの現場の時刻を修正 |

### データのバックアップ

> [!CAUTION]
> 本システムの全データは Supabase 上に保存されています。Supabase のダッシュボードから定期的にバックアップを取ることを推奨します。

### サポート

技術的な問題が発生した場合は、ブラウザの開発者ツール（F12）のConsoleタブでエラーメッセージを確認し、状況を記録のうえ管理者へご連絡ください。
