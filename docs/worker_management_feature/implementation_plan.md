# 作業員管理と実績集計の追加機能 計画書

ユーザーからの要望に基づき、管理画面（AdminApp）に「作業員」タブを追加し、「①作業員マスター管理機能」と「②作業員別の稼働実績・集計機能」を実装するための計画です。

## 概要
現在 `Workers` テーブルに保存されている作業員情報をAdminAppから直接管理できるようにします。また、これまで現場単位（Projects）で集計していた実績データを、作業員（Worker）単位で集計・表示する画面を追加します。

## 提案する変更内容

### 1. `src/AdminApp.jsx` の修正

#### [MODIFY] `src/AdminApp.jsx`
*   **「作業員」タブの追加:**
    *   画面上部のナビゲーションタブ（ホーム、管理シート、実績入力、工事設定）に、新たに `workers`（作業員）を追加します。
*   **機能①: 作業員マスター管理機能の実装:**
    *   `activeTab === 'workers'` の場合に表示されるUIを構築します。
    *   **一覧表示・編集**: 現在の `workers` ステートのリストを表示し、それぞれの名前をインラインで編集・保存できるようにします（`supabase.from('Workers').update`）。
    *   **追加・削除**: 新規の作業員を追加する機能と、削除（または論理削除）する機能を実装します。
    *   **表示順変更**: リスト上で「↑」「↓」ボタンを押すことで `display_order` を入れ替え、WorkerApp（実績入力アプリ）側のログイン画面の表示順を制御できるようにします。
*   **機能②: 作業員別実績・集計機能の実装:**
    *   作業員タブ内に「マスター管理」領域と「稼働集計」領域を設けます（またはサブタブで切り替え）。
    *   AdminAppで読み込んでいる全実績データ（各プロジェクトの `records`等）を集約し、「各作業員が・どの現場で・何時間稼働したか」を一覧できるテーブル（またはリスト）を作成します。
    *   全体の労働時間の合計（累計h）も表示します。
*   **アイコンのインポート追加:**
    *   `lucide-react` から `Users`, `ArrowUp`, `ArrowDown` などの必要なアイコンを追加インポートします。

### 2. その他の確認事項
*   既存の `WorkerApp.jsx` は、開始時に `Workers` を `display_order` 順にフェッチしているため、AdminApp側で `display_order` を更新すれば即座に反映される想定です。

2.  **マスター管理の確認:**
