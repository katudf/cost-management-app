# 作業員タブ（マスター管理・集計機能）の動作確認手順

管理画面（AdminApp）に「作業員」タブを追加し、作業員マスターの設定と稼働実績の確認ができるようになりました。以下の手順で動作を確認してください。

## 1. タブの表示確認
1.  開発サーバーの画面（PC向けのAdminApp）を開きます。
2.  画面上部のナビゲーションタブに**「作業員」**が追加されていることを確認します。
3.  「作業員」タブをクリックし、画面が切り替わることを確認します。

## 2. 作業員マスター設定の確認（画面左側）
1.  **一覧表示:** 現在登録されている作業員がリスト表示されていることを確認します。
2.  **名前の編集:** 任意の作業員の名前（テキストボックス）を書き換え、枠外をクリック（またはEnter）して名前が保存されることを確認します。
    *   *※変更後、ページをリロードしても変更された名前が維持されていれば成功です。*
3.  **表示順の変更:** リストの左側にある「↑」「↓」ボタンをクリックし、作業員の並び順を入れ替えます。
    *   *※変更後、ページをリロードして順序が維持されていること。また、スマホ側の作業員アプリ（`/worker`）のログイン画面でも順序が反映されているか確認してください。*
4.  **新規追加:** 右上の「追加する」ボタンを押して新しい作業員（例：「テスト作業員」）を追加し、リストの一番下に表示されることを確認します。
5.  **削除:** 追加した「テスト作業員」の右側にあるゴミ箱アイコン（削除）をクリックし、リストから消えることを確認します。

## 3. 稼働時間集計の確認（画面右側）
1.  画面右側のテーブルに、各作業員の名前と「稼働した現場」「総稼働時間」が表示されていることを確認します。
2.  「総稼働時間」が、これまで入力された全日報の合計時間と一致している（またはそれらしい実数値になっている）ことを確認します。
    *   *※まだ日報がない作業員の部分はテーブルに表示されない、または0hになります。*
3.  **リアルタイム反映確認:**
    *   AdminAppの「実績入力」タブ、またはWorkerAppから、任意の作業員で適当な実績（例：1時間）を新たに登録します。
    *   「作業員」タブに戻り、その作業員の「総稼働時間」が先ほど入力した分（+1時間）だけ増えていることを確認します。

特に問題なく動作しましたら、タスク完了となります。
