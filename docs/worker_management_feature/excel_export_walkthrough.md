# 作業員個人別「就労日報」Excel出力機能の動作確認手順

以下の手順に従って、機能が正しく実装されているか検証してください。

## 1. データベースへのカラム追加（まだ行っていない場合）
`docs/worker_management_feature/schema.sql` の内容をコピーし、SupabaseのSQL Editor等で実行して、`TaskRecords`テーブルに `overtime_hours` カラムを追加してください。

## 2. WorkerApp（スマホ側）での早出・残業入力テスト
1.  PCまたはスマホから WorkerApp（`/worker`）を開き、適当な作業員でログインします。
2.  適当な現場を選び、実績入力画面を表示します。
3.  各作業項目の入力枠の下に「うち時間外稼働 (早出・残業)」の入力枠（オレンジ色）が追加されていることを確認します。
4.  通常稼働時間（例：`8.0`h）と、時間外稼働（例：`2.0`h）をそれぞれ入力し、日報を送信（保存）します。
5.  ※ページをリロードしても保存した時間が維持されているか確認してください。

## 3. AdminApp（PC側）でのExcel出力テスト
1.  PCから AdminApp（管理画面）を開きます。
2.  「作業員」タブを開き、先ほど実績を入力した作業員の行の右端にある「出力」ボタンをクリックします。
3.  モーダル画面が表示されるので、出力する週（直近の月曜日）を選択し、「Excel出力」ボタンを押します。
4.  ダウンロードされたExcelファイルを開き、以下の点を確認します。
    *   **タイトルと名前:** 「就労日報 (R8)」と「作業者名」が正しく出力され、枠が結合されているか。
    *   **日付と曜日:** 選択した週の月〜日の日付と曜日が横に並んでいるか。
    *   **実績の反映:** 先ほど入力した日の列に、現場名、時間（例:`8h\n(+外2h)`）、作業内容が反映されているか。
    *   **手当時間の合計:** 画面下部の「作業手当」の「時間(H)」の枠に、入力した残業時間の合計（例:`2H`）が表示されているか。

## 4. 職長としての「協力会社」出力テスト（オプショナル）
1.  AdminAppの「工事設定」タブで、現在入力した現場の「担当職長」を、いま操作している作業員に設定します。
2.  WorkerAppにて、職長としてその現場の実績を入力し、**協力会社（例：〇〇塗装 2名）** を入力して送信します。
3.  再度AdminAppの「作業員」タブからExcelを出力します。
4.  Excelの下部「協力会社」の枠に、「〇〇塗装 ( 2名 )」と自動印字されているか確認します。
