# 配置予定表：ガントチャートの直感的なドラッグ＆リサイズ機能の実装プラン

配置予定表上部に表示されている「案件バーチャード」に対して、マウス操作による直感的な工期変更機能を実装します。

## ユーザー承認が必要です (User Review Required)

以下の主要な設計についてご確認ください。問題がない場合は実装を開始します。

> [!IMPORTANT]
> **1. ドラッグ操作による3つのモード**
> ガントバーに対して以下の３つの操作を可能にします。
> - **バー左端のドラッグ (伸縮)**: `startDate` (開始日) のみを変更します（終了日は固定）。
> - **バー右端のドラッグ (伸縮)**: `endDate` (終了日) のみを変更します（開始日は固定）。
> - **バー中央部のドラッグ (移動)**: 開始日と終了日が移動しますが、指定された要件に従い「**休日（土・日・会社休日）を除いた実働日数**」が変わらないように、終了日を自動計算して維持します。
> 
> ※ 休日をまたぐように移動した場合は、自動的に平日分が伸び縮みし、カレンダー上のバーの見た目の長さ（日数）は変わる場合がありますが、実働人工の計算に影響しないように保たれます。

> [!WARNING]
> **2. 変更の即時反映と保存**
> ドラッグ操作中（マウスを動かしている最中）は、一時的にバーがプレビュー表示として動きます。マウスを離した瞬間に、Supabase（`Projects`テーブル）へ更新リクエストが送信され、確定します。

## 提案する変更内容

### [MODIFY] `src/components/tabs/AssignmentChartTab.jsx`
1. **休日判定ロジックの強化**
   - 既存の `companyHolidays` とカレンダー（土・日）を組み合わせた `isHoliday(dateObj)` 関数を実装。
   - `countWorkingDays(start, end)`: 期間内の稼働日数を計算。
   - `addWorkingDays(start, workingDaysCount)`: 開始日から指定した稼働日数分だけ進めた終了日を計算。
2. **ドラッグ状態の管理ステートの追加**
   - `[draggingGantt, setDraggingGantt]` : 現在ドラッグ中のプロジェクトID、モード（left, right, move）、ドラッグ開始位置、ドラッグ開始時の日付情報を保持。
3. **UI/イベントリスナーの実装**
   - プロジェクトバーに3つの操作領域（左リサイズ用、中央移動用、右リサイズ用）を持たせたHTML構造へ変更。
   - `onPointerDown` でドラッグ開始を変数にキャプチャ。
   - `onPointerMove` で（48px単位の）移動量を計算し、プレビュー用のバー情報を一時生成して画面に描画。
   - `onPointerUp` で計算結果をDBに保存し、マスターの `barProjects` ステートを更新。
