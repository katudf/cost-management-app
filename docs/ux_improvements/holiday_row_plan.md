# 実装計画: 会社行事/休日行でのイベント種別登録機能

ご要望の仕様を実現するための実装計画です。

## データベースの更新について
**データベースのテーブル追加や新しいカラムの追加は必要ありません。**
現在の `CompanyHolidays` テーブルにはすでに `description` というテキスト保存用のカラムが存在します。今回は、画面上で選択した「会議」や「社員旅行」といった文字をこの `description` カラムに保存・上書きするだけで実現できます。

## Proposed Changes

### `AssignmentChartTab.jsx`
1. **状態の拡張**:
   - 現在は「日付の配列(`['2026-04-01', ...]`)」として保持している `companyHolidays` を、「オブジェクトの配列（日付と内容のセット: `[{ date: 'xxx', description: '会議' }, ...]`）」に変更し、`description` を画面で扱えるようにします。
2. **セルクリック処理の実装**:
   - 新しく追加した「会社行事/休日」行の各日付セルをクリック可能（`onClick`処理追加）にします。
   - クリックされた際、ポップアップ（または専用の選択UI）を表示します。
3. **登録用ポップアップの追加**:
   - 「休日」「会議」「社員旅行」「削除」を選択できる小さなメニューを表示し、選択された内容を `CompanyHolidays` テーブルに `Insert/Update` （または削除）します。
4. **セルの描画更新**:
   - 登録されたイベント（例:「会議」）に応じて、セルの背景色や表示テキスト（「会議」や「旅行」など）を切り替えて表示します。
