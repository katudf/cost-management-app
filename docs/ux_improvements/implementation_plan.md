# 画面の切り替えと文脈の分断解消プロジェクト

配置表（AssignmentChartTab）を中心としたUX改善案について、実装難易度が低い順から段階的に実装する計画です。

## User Review Required
以下のプランで順次実装を進めてもよろしいでしょうか？（今回はフェーズ1とフェーズ2の「比較的単純だが効果の高いもの」を優先して実装します）

## フェーズ1: 現場名ホバーによるサマリーツールチップ（難易度: 低）
プロジェクトの進捗率や予測利益などの計算済み結果を配置表に引き渡し、現場名にマウスを乗せた際にサマリーをポップアップ表示します。

### AdminApp.jsx
- `AssignmentChartTab` に対して、計算済みの各種サマリーが入っている `dashboardStats.displayProjects` を渡すように変更します。

### AssignmentChartTab.jsx
- `onMouseEnter` と `onMouseLeave` を使った独自のツールチップ用ステートを追加します。
- 現場名（左側の固定列）にカーソルを乗せたとき、その現場の「進捗率」「予測粗利」「消化工数/目標工数」をポップアップ表示するUIコンポーネントを追加します。

## フェーズ2: ポップアップからの「詳細画面ジャンプ機能」（難易度: 低〜中）
配置の追加・変更・削除を行う小さなポップアップ内に、現場の詳細設定画面へワンクリックで飛べるショートカットボタンを追加します。現状の設計を変えずに、最も素早く現場詳細へアクセスできる手法です。

### AdminApp.jsx
- `AssignmentChartTab` に対して、タブ移動と対象プロジェクト設定を行うための関数 `setActiveTab`, `setActiveProjectId` を連携します。

### AssignmentChartTab.jsx
- ポップアップ内の「現場」一覧および「現在の配置」リストの各現場の横に `[詳細設定を開く]` ボタンを追加します。
- ボタンクリック時に、選んだ現場のIDをセットしつつ `master` タブ（工事詳細）へ直接遷移させます。

---

> [!NOTE]
> 「1. クイック詳細パネル（ドロワー）」は、既存の非常に複雑なMasterTab（詳細設定パネル）を完全に切り出して使い回せるようにする「大規模なコンポーネントの分割作業」が必要になるため、**難易度: 中〜高**となります。
> まずはフェーズ1、フェーズ2を実装して使用感を確かめていただき、それでも不便であればドロワーへの改修を後日行うアプローチを推奨します。
