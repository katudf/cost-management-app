# 実装計画: 1セルから複数セルへのペースト拡張

## Goal
現在の「選択範囲からのペースト」機能を拡張し、**「1日分のスケジュールをコピー（またはカット）し、複数日をドラッグ選択してペーストした際、その複数日すべてにスケジュールを展開する」**機能を追加します。
これにより、特定の1日の布陣を、ドラッグで選択した1週間すべてに一気に割り振ることが可能になります。

## 動作の仕様 (スプレッドシート準拠)
ペースト時 (`Ctrl + V`) に、以下の判定ロジックを追加します。

1. **「1セル→複数セル」の判定**
   - クリップボードに保持されているデータが「単一の日付からのもの（`dayOffset` がすべて 0）」である。
   - ペースト先の選択範囲（`editCell.dragDates`）が「複数（2日以上）」である。
   上記2点を満たす場合、「1対多のペースト」モードとして動作します。

2. **データの展開方法**
   - 「1対多」モードの場合：クリップボードにある1日分の配置データを、選択された**すべての日付**に対して一括で複製（Insert）します。
   - それ以外（複数日をコピーして複数日にペースト等）の場合：従来通り、選択範囲の「開始日」を起点として、元の相対日数の間隔を保ったままペーストします。

3. **カット＆複数ペーストの挙動**
   - カット(`Ctrl+X`)を行った場合は、カット時に元のデータが削除されているため、ペースト時の挙動はコピー時と同様（選択した複数日に展開される）になります。スプレッドシートでは「カット→複数ペースト」はエラーになったり単一セルにのみ貼られたりしますが、本システムではご要望に合わせ「複数日に展開する」仕様で統一します。

## Proposed Changes
### `AssignmentChartTab.jsx`
- `handleKeyDown` の `Ctrl + V` のブロックを修正します。
- クリップボードデータ内から「最大日付オフセット」を算出し、単一日か複数日かを判定。
- ペースト先が複数日かつ、単一日のコピーデータである場合、`targetDates.forEach`を用いて、選択されたすべての日付に対して `newRecords` を生成するようにループ処理を分岐させます。
