# 確認リスト：エクセルインポートの重複名制御

## 実装内容
- 同じ名前の現場が存在するエクセルを読み込んだ場合、「既に登録されています」という警告が出るようにしました。
- 重複時は以下の2つのアクションのみが選択できるようにしました。
  1. **既存の現場に上書き**: 既存の重複プロジェクトの作業項目を、読み込んだエクセルのデータで上書きします。
  2. **別名で新規登録**: 別の名前（デフォルトで「元の名前 (コピー)」）を入力し、新しい現場として登録します。

## ユーザー検証のお願い
以下の手順で、意図したとおりに動作するかご確認をお願いします。

1. **準備**: アプリ内で任意の現場（例：「テスト現場」）を作成する。
2. **インポート**: 工事名が「テスト現場」になるエクセルファイル（またはファイル名が「テスト現場.xlsx」のもの）をインポートする。
3. **重複時のモーダル確認**: 「テスト現場 は既に登録されています。」という警告と、「上書き」「別名で新規登録」のメニューが表示されることを確認する。
4. **別名登録のテスト**: 
   - 「別名で新規登録」を選び、名前を「テスト現場 (別案)」等に変更して「この名前で登録する」を押す。
   - 新しい現場として追加され、アクティブになることを確認する。
5. **上書きのテスト**:
   - 再度同じファイルをインポートする。
   - 今度は「既存の現場に上書き」を選択する。
   - 既存の「テスト現場」のマスターデータ（作業項目）が更新され、アクティブになることを確認する。

## 検証結果
- 同名現場名（＋「(コピー)」）の別名で新規登録できることを確認済み。
- （既存現場への上書き更新も正常に動作する設計通りとなっています。）
