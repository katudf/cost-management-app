# エクセルインポートの重複名制御

## 概要
エクセルからのデータインポート時に、読み込んだ工事名がすでに登録されている現場名と一致した場合、既存データを上書きするか、別名で新規登録するかをユーザーが選択できるようにします。

## 修正内容

### App.jsx
既存の `handleExcelImport` と `handleImportChoice`、およびインポートモーダルのUIを以下のように修正します。

#### [MODIFY] App.jsx
- **状態の追加**:
  - `aliasName` (別名入力用ステート)
- **handleExcelImport の修正**:
  - インポートしようとする現場名が、既存の `projects` の `siteName` と一致するかチェック。
  - 一致する場合、`importModalInfo` に `type: 'duplicate'` をセットし、対象のプロジェクトIDを保持させる。
  - 一致しない場合は従来通りのフロー (`type: 'normal'`) とする。
- **インポート時のモーダルUIの修正**:
  - `importModalInfo.type === 'duplicate'` の場合、以下の2つの選択肢を表示する。
    1. **既存の現場に上書き**: 一致した既存プロジェクトの作業項目（masterData）を今回のデータで上書きする。
    2. **別名で新規登録**: 新しい名前（デフォルトは「元の名前 (コピー)」）を入力するテキストボックスと登録ボタンを表示し、別名の新規プロジェクトとして作成する。
- **handleImportChoice の修正**:
  - `overwrite_duplicate` の場合、対象プロジェクトの `masterData` を更新し、そのプロジェクトをアクティブにする。
  - `create_alias` の場合、入力された `aliasName` を用いて新規プロジェクトを作成し、アクティブにする。

## 検証計画
### 手動検証
1. ブラウザでアプリケーションを開く。
2. 任意の現場（例：「テスト現場」）を作成し、適当なデータを設定する。
3. エクセルファイル（工事名が「テスト現場」と判定されるもの、またはファイル名が「テスト現場」のもの）をインポートする。
4. 重複警告モーダルが表示され、「既存の現場に上書き」と「別名で新規登録」の選択肢が出るか確認する。
5. 「別名で新規登録」を選び、名前を変更して登録できるか確認する（別の現場として追加されるか）。
6. 再度インポートし、「既存の現場に上書き」を選んだ場合、元の現場のデータが上書きされ、アクティブなプロジェクトが切り替わるか確認する。
