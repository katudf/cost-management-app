# イレギュラー入力ケース対応 実装計画

シミュレーションレポートで指摘された各種イレギュラーについて、日跨ぎ（夜間）対応を含む入力制限と計算ロジックの改善を行います。

## ユーザーレビューが必要な事項

> [!IMPORTANT]
> - 「日跨ぎ（翌日）」を示すチェックボックスを追加します。これにチェックを入れると「開始時間が終了時間以降」の入力が許可され、終了時間に+24時間して計算されます。
> - 今後、片方の時間しか入力せずに送信ボタンを押下した場合、「実働時間0h」として無視されるのではなく、**入力エラーとして送信がブロック**されるようになります。

## 提案される変更

### 1. `src/utils/workTimeUtils.js`

#### [MODIFY] calculateWorkHours
- 第4引数として `isOvernight` フラグを受け取るよう改修します。
- `isOvernight = true` の場合、終了時間を表す分（`endMin`）に +1440分（24時間）を加算してから労働時間を計算します。

### 2. `src/WorkerApp.jsx`

#### [MODIFY] UIレイアウトおよび時間枠のState
- `tasks` 内の各 `time_slots` に `is_overnight` プロパティを追加します。
- 既存のDBデータをフェッチしてマッピングする際、すでに `start_time > end_time` となっている実績があれば自動的に `is_overnight = true` として画面に展開します。（DBスキーマの変更は不要です）
- 画面上で各時間入力枠の隣に `[ ] 翌日` などのチェックボックスUIを追加します。

#### [MODIFY] timeOverlapWarnings（時間重複チェック）
- バリデーションにおいて、`is_overnight == true` の場合は逆転時間（`start >= end`）をエラーとせず、通常の時間重複計算においても `end` を +1440 して正しく判定されるように修正します。

#### [MODIFY] handleSubmit（送信前バリデーション）
以下の条件いずれかに該当する場合、送信を中断し `showToast` で適切なエラーメッセージを出します。
1. **片方未入力**: 開始時間と終了時間のどちらかに値が入っているのに、もう一方が空欄の場合。
2. **異常な長時間労働**: 計算された `netWorkHours` が異常に長い（例：20時間以上）場合。
3. **協力業者の不正な人数**: 会社名が入力されているが、人数が「0以下」または「小数」の場合。

## 検証プラン

### 手動確認
1. **日跨ぎのチェック**: 夜間作業（20:00〜05:00等）を入力し、「翌日」チェックを入れずに送信ボタンを押すと警告が出ることを確認。チェックを入れると実働時間が正しく計算（例：休憩控除で約8時間）され、送信できることを確認する。
2. **片方未入力**: 08:00〜(空欄) で送信するとブロックされることを確認する。
3. **協力業者**: 会社名を入れて人数を `-1` や `0` にした場合にブロックされることを確認する。
