# 作業員詳細情報表示UI実装 Walkthrough (モーダル版)

## 変更内容の概要
作業員の一覧行をクリックした際、インラインでの展開（アコーディオン形式）ではなく、新しく作成した別モーダルで詳細情報を表示するようにUIを変更しました。

## 実装の詳細
1. **`WorkerDetailsModal` コンポーネントの作成**
   - 既存の「作業員情報の編集」画面 (`WorkerEditModal.jsx`) のデザインレイアウトに合わせて、各項目（氏名、ふりがな、生年月日など）が視覚的にわかりやすく整理された情報を表示する画面を実装しました。
   - モーダルの右下に「閉じる」「編集する」「削除する」のアクションボタンを配置し、詳細情報を見ながらすぐに操作できるようにしました。

2. **`WorkersTab.jsx` の構成変更**
   - アコーディオン表示に関するロジックを廃止し、行をクリックした際に `selectedWorkerForDetails` に該当の作業員IDをセットしてモーダルを呼び出す仕組みに変更しました。
   - 一覧の各行に「詳細」テキストと右向きの矢印（`ChevronRight`）を追加し、クリックできる要素であることをより直感的に示しました。

## 完了した確認事項
- [x] 作業員リストの行をクリックすると、詳細情報モーダルが独立して開くこと。
- [x] モーダルのデザインが「編集画面」と統一感のあるレイアウトになっていること。
- [x] モーダル内のアクションボタンで、表示・編集・削除が正しく切り替わること。
