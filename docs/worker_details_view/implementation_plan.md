# 作業員詳細情報表示機能の実装計画

## 目的
作業員マスター設定（`WorkersTab.jsx`）において、現在のホバーによる編集・削除ボタンの表示を廃止し、作業員名（または行全体）をクリックした際に詳細情報が展開（アコーディオン形式）されるようにUIを変更する。詳細情報エリア内に編集と削除のボタンを配置する。

## 提案される変更箇所
### UI (components/tabs/WorkersTab.jsx)

- **状態管理の追加:**
  - `import React, { useState } from 'react';` を追加。
  - `const [expandedWorkerId, setExpandedWorkerId] = useState(null);` をコンポーネント内に追加し、展開されている作業員のIDを管理する。

- **リストの行（コンテナ）の変更:**
  - 各作業員の行（div要素）に `onClick` イベントを追加し、クリックで `expandedWorkerId` をトグル切り替え（開閉）できるようにする。
  - 既存の「順番入れ替えボタン（↑・↓）」のクリックイベントには `e.stopPropagation()` を追加し、意図せず行が展開・折り畳みされるのを防ぐ。
  - マウスオーバー時にカーソルがポインターになるように `cursor-pointer` クラスを追加する。

- **ホバーアクションの削除:**
  - 既存の右端にある、ホバー時に表示される編集 (`openEditWorkerModal`) および削除 (`removeWorker`) ボタン群を削除・移動する。

- **展開エリア（アコーディオンの中身）の実装:**
  - `expandedWorkerId === worker.id` の場合のみ表示される詳細情報エリアを行の下部に追加する。
  - 詳細情報として以下を表示する。
    - フリガナ、年齢、連絡先などの詳細情報（現在隠れがちな情報を含む）。
  - 詳細情報エリアの右下に、前述の「編集」と「削除」のボタンを配置する。

## 懸念事項や確認事項（ユーザーレビュー必須）
- [CAUTION] UIはアコーディオン形式（インラインでの展開）を想定していますが、別モーダルで表示する方がご希望である場合はその旨お知らせください。
- 作業員を入れ替えるための上下ボタン（矢印）はクリックしてもアコーディオンが開閉しないように調整します。

## 検証計画
- 作業員タブを開き、各作業員をホバーした際に編集・削除ボタンが表示されないことを確認。
- 作業員名（枠内）をクリックすると、詳細エリアがスライドダウンし、その中に詳細情報および編集・削除ボタンが表示されることを確認。
- 編集ボタン、削除ボタンがそれぞれ正常に機能することを確認。
- 順番の並び替え（↑↓ボタン）を操作した際に、詳細情報の開閉が誤動作しないことを確認。
