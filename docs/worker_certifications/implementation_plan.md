# 資格情報の表示追加 実装計画

## 目的
作業員の詳細情報モーダルにおいて、保持している資格情報（`WorkerCertifications`）を一覧表示できるようにする。

## DBスキーマ構成の確認
調査の結果、`WorkerCertifications` テーブルは以下のようなカラムを持っていることを確認した。
- `id`
- `created_at`
- `workerId`
- `name` (資格名)
- `acquisitionDate` (取得日)
- `expiryDate` (有効期限)

## 提案される変更箇所
### 1. データアクセス (AdminApp.jsx)
起動時のデータ取得 (`fetchAllData` 関数内) にて、`WorkerCertifications` も併せて取得する。
取得した資格情報を `workerId` に基づいて `workers` 配列の各要素の `certifications` プロパティにマッピングしてステートに保存する。

### 2. UI表示 (WorkerDetailsModal.jsx)
「作業員詳細情報」モーダルの末尾近辺に「資格情報」セクションを追加する。
- 資格が存在する場合はリスト形式（資格名、取得日などのバッジ表示）で一覧を展開する。
- 資格がない場合は「登録されている資格情報はありません」といったプレースホルダー文言を表示する。

## 懸念事項・確認事項
- 資格情報の編集（追加・削除など）は今回はスコープ外とし、あくまで**「表示」**のみを実装する前提とする。
- 実行時のレスポンス改善のため、初回データのロード時に全作業員の資格情報を一括取得する方針とする。
