# 天気予報・ニュース表示スリム化 実装計画書

本計画書は、横画面時の物理的な高さ制限（360dp）において、天気予報およびニュースの全情報が見切れることなく美しく表示されるよう、UIレイアウトをスリム化する方針を定義します。

## 1. ゴール説明
- 右半分の情報エリア（天気およびニュース）のUIを最適化し、縦スクロールなしで全体の表示が画面に綺麗に収まるようにする。
- プレミアムなダークモードデザイン、適切なコントラスト、情報の視認性を維持する。

## 2. 影響分析（自己検証）

### 設計ポリシー（Design Policies）との整合性
- **デザインのプレミアム性**: 黒背景を基調とし、アクセントカラー（降水確率の水色系、温度の青/赤など）を用いて視覚的に整理された、スタイリッシュなカードレイアウトを採用します。
- **画面全体の調和**: 余白（padding）やフォントサイズ、週間予報の表示数を調整して360dpの画面に美しく収めます。

### 重要経路（Critical Paths）への副作用
- **非同期処理の維持**: UIレイアウトの調整のみのため、GPSによる位置情報取得やバックグラウンドでの天気・ニュースの定期通信（コルーチン）には一切影響しません。

### デグレード（先祖返り）の防止
- **フォールバック処理**: 週間予報を `take(3)` で制限する際や、データがない場合のフォールバック（空リストなど）でもクラッシュしないよう安全にComposeを描画します。また、プレビュー用ダミーデータも同期させてビルドエラーを防ぎます。

## 3. 提案する変更内容

### 3.1. UIレイヤーの変更
#### [MODIFY] [MainScreen.kt](file:///c:/Users/katuy/Desktop/cost-management-app/Standby_app/app/src/main/java/com/example/standbyclockapp/ui/main/MainScreen.kt)
- **情報パネルの1つのカードへの統合と左右分割レイアウト**:
  - `WeatherSection` と `NewsSection` を別々のカードとして描画するのをやめ、右側の情報エリア全体を1つの「インフォメーションカード」としてまとめます。
  - さらに、天気情報の高さを圧縮するため、現在の天気を1行に集約し、「時間別予報」と「週間予報」を縦に並べるのではなく左右に分割（Row）して配置します。
  - これにより、物理画面の高さ制限（実質 320〜350dp）において、確実に見切れることなく天気情報とニュースの両方を表示可能にします。

- **`MainDisplay`**:
  - 右側の `Column` 内に、背景・枠線付きの1つの統合カード（`InformationCard`）を配置します。
  - 縦パディングを `12.dp` に設定。

- **統合された `InformationCard` の構成**:
  1. **現在の天気**:
     - 地名、天気説明、絵文字、気温、降水確率を1行のコンパクトな Row に集約します。
  2. **予報エリアの左右分割レイアウト**:
     - 左側：時間別予報（直近3時間）を縦並び（Column）で表示。
     - 右側：週間予報（直近3日間）を縦並び（Column）で表示。
     - これらを Row で並列に配置し、縦のスペースを約 70dp 削減します。
  3. **区切り線（Divider）**:
     - 薄い白色の水平線（高さ 1.dp、アルファ値 0.1）。
  4. **ニュースフィード（スライドショー）**:
     - カードの下部に直接配置。
     - 見出し「ニュース」は `11.sp` の薄いテキストとして配置し、ニュースタイトルは `13.sp` で最大2行表示（高さ 36.dp に制限）。

## 4. 検証計画

### 4.1. 手動・エミュレータによる検証
1. アプリをビルドしてエミュレータに展開する。
2. 右半分が1つの美しい統合カードになり、天気情報とニュースフィードが両方とも見切れずに収まっていることを確認する。
3. `adb shell screencap` を利用してスクリーンショットを取得し、最終的なレイアウトを検証する。
